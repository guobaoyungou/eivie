# API管理功能部署和使用指南

## 功能概述

后台统一API菜单管理功能已成功实现，提供以下核心能力：

### 主要功能
- ✅ **接口列表管理** - 展示所有API接口，支持分类、搜索、筛选
- ✅ **接口自动扫描** - 自动扫描控制器，智能识别API接口
- ✅ **接口详情编辑** - 完善接口信息，添加参数说明和示例
- ✅ **在线测试工具** - 直接在后台测试API接口，查看响应
- ✅ **测试历史记录** - 记录所有测试操作，便于审计和调试
- ✅ **权限控制** - 仅平台管理员可访问

## 部署步骤

### 第一步：导入数据库表结构

```bash
# 进入数据库
mysql -u用户名 -p

# 选择数据库
use 你的数据库名;

# 导入表结构
source /www/wwwroot/eivie/database/migrations/api_management_tables.sql;

# 导入初始数据（可选）
source /www/wwwroot/eivie/database/migrations/api_auth_interfaces_data.sql;

# 查看是否导入成功
show tables like 'dd_api_%';
```

或者使用命令行一行执行：
```bash
mysql -u用户名 -p密码 数据库名 < /www/wwwroot/eivie/database/migrations/api_management_tables.sql
mysql -u用户名 -p密码 数据库名 < /www/wwwroot/eivie/database/migrations/api_auth_interfaces_data.sql
```

**注意事项：**
- 初始数据SQL中的`aid`默认为1，请根据实际情况修改
- 如果表已存在，会跳过创建（使用了IF NOT EXISTS）

## 使用指南

### 1. 首次使用 - 扫描接口

#### 操作步骤：
1. 点击菜单：**API > 接口扫描**
2. 选择扫描类型：
   - **全量扫描**：扫描所有Api开头的控制器
   - **增量扫描**：只扫描新增的接口
3. 勾选要扫描的控制器（建议全选）
4. 点击"开始扫描"按钮
5. 查看扫描结果（新增接口和需更新的接口）
6. 点击"保存接口"按钮，将接口信息保存到数据库

#### 扫描规则：
- 自动识别`app/controller`目录下所有以`Api`开头的控制器
- 解析控制器中的public方法
- 从方法注释中提取接口信息
- 根据控制器名称自动分类

#### 自动识别的分类：
- ApiIndex → 用户认证
- ApiAdminMember → 会员管理
- ApiAdminOrder → 订单管理
- ApiAdminProduct → 商品管理
- ApiAdminFinance → 财务管理
- AiTravelPhotoScene → AI旅拍-场景
- AiTravelPhotoOrder → AI旅拍-订单
- 等等...

### 2. 查看接口列表

#### 页面布局：
```
┌─────────────┬──────────────────┐
│  左侧分类树  │   中间接口列表    │
│             │                  │
│ 全部接口    │  [搜索框] [筛选] │
│ 用户认证    │                  │
│ 会员管理    │  ┌─────────────┐ │
│ 订单管理    │  │ GET 用户登录 │ │
│ ...         │  └─────────────┘ │
└─────────────┴──────────────────┘
```

#### 功能说明：
- **左侧分类树**：点击分类快速筛选接口
- **搜索框**：支持按接口名称或路径搜索
- **筛选器**：
  - 按请求方式筛选（GET/POST/PUT/DELETE）
  - 按认证要求筛选（需要登录/无需登录）
- **接口卡片**：显示接口的基本信息
  - 请求方式标签（不同颜色）
  - 接口名称和路径
  - 简要描述
  - 标签信息

#### 操作：
- 点击接口卡片：查看详细信息
- 右上角"扫描接口"按钮：重新扫描

### 3. 编辑接口信息

#### 可编辑的字段：
**基本信息：**
- 接口名称（必填）
- 接口分类
- 认证要求（需要登录/无需登录）
- 状态（启用/停用）
- 排序（数字越大越靠前）

**接口描述：**
- 详细的接口说明文档

**请求参数：**
- JSON格式定义参数列表
- 包含：参数名、类型、是否必填、说明

**响应示例：**
- JSON格式的响应数据示例

**其他信息：**
- 标签（多个标签用逗号分隔）
- 备注说明

#### 参数定义示例：
```json
[
  {
    "name": "aid",
    "type": "int",
    "required": true,
    "desc": "账户ID"
  },
  {
    "name": "keyword",
    "type": "string",
    "required": false,
    "desc": "搜索关键词"
  }
]
```

#### 响应示例格式：
```json
{
  "status": 1,
  "msg": "成功",
  "data": {
    "id": 12345,
    "name": "测试数据"
  }
}
```

### 4. 在线测试接口

│ 左侧参数输入 │ 右侧响应展示  │

5. 右侧显示响应结果：
   - 状态码（绿色=成功，红色=失败）
   - 响应时间（毫秒）
   - 响应数据（JSON格式高亮显示）

#### 特性：
- 自动根据参数定义生成表单
- 必填参数标红提示
- JSON响应自动格式化和语法高亮
- 测试记录自动保存到日志

### 5. 查看测试历史

#### 访问方式：
- 菜单：**API > 测试历史**

#### 显示内容：
- 测试ID
- 接口名称和路径
- 状态码
- 响应时间
- 请求IP

#### 操作：
- 点击"查看详情"：查看完整的请求参数和响应数据
- 可用于复现问题或验证历史测试

## 权限说明

### 访问权限
- ✅ **平台管理员**：完全访问权限
  - isadmin = 2 的用户
  - aid = 0 的系统管理员
- ❌ **商户用户**：不显示API菜单
  - bid > 0 的用户
- ❌ **权限配置模式**：不显示API菜单
  - uid = -1 时（编辑权限配置时）

### 菜单权限标识
```
ApiManage/* - API管理总权限
  ├─ ApiManage/index - 接口列表

  ├─ ApiManage/savescan - 保存扫描结果

  ├─ ApiManage/testlog - 测试历史
  ├─ ApiManage/testlogdetail - 测试日志详情

```

## 常见问题

### Q1: 菜单不显示怎么办？
**A:** 检查以下几点：
1. 确认使用平台管理员账号登录
2. 清理缓存：`rm -rf runtime/cache/*`
3. 检查Menu.php文件是否正确修改
4. 确认不是在权限配置模式（uid != -1）

### Q2: 扫描不到接口？
**A:** 可能的原因：
1. 控制器名称不是以`Api`开头
2. 方法不是public
3. 控制器文件不在`app/controller`目录
4. PHP反射权限不足

### Q3: 测试接口失败？
**A:** 检查：
1. 参数是否填写完整
2. 接口路径是否正确
3. 服务器是否可以访问自己的域名
4. 查看测试日志获取详细错误信息

### Q4: 如何导出API文档？
**A:** 
1. 在接口列表勾选要导出的接口
2. 点击"导出文档"按钮
3. 选择格式（Markdown或JSON）
4. 下载生成的文档文件

**注意**：当前版本的导出功能在detail.html页面中，如需批量导出，可以在index.html页面添加批量操作功能。

### Q5: 如何添加自定义分类？
**A:** 修改`ApiManageService.php`中的`getCategoryByController`方法，添加新的分类映射规则。

### Q6: 数据库表前缀不是dd_怎么办？
**A:** 需要修改：
1. SQL文件中的表名前缀
2. 控制器和服务层中的表名引用
