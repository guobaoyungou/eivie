# 统一订单管理功能快速上手指南

## 一、功能简介

统一订单管理功能将平台所有订单类型（商城、拼团、秒杀、团购、砍价、幸运拼团、积分兑换、预约、课程、周期购、选片等）聚合到一个页面中展示，用户可以一站式查看所有消费记录。

## 二、核心文件说明

### 后端文件

| 文件路径 | 说明 |
|---------|------|
| `/app/controller/ApiUnifiedOrder.php` | 统一订单聚合控制器（核心） |

### 前端文件

## 三、如何新增订单类型

假设要新增"拍卖订单"类型，标识为 `auction`，数据表为 `auction_order`：

### 步骤1：配置订单类型（后端）


### 步骤2：配置详情页路径（后端）

在同一文件中：

```php
protected $detailRoutes = [
    // ... 现有路径

];
```

在同一文件中添加方法（可参考现有方法）：

```php

        'status'           => (int)($row['status'] ?? 0),
        'status_text'      => $this->statusText($row['status'] ?? 0),
        'item_count'       => 1,
        'create_time'      => $timestamp ? date('Y-m-d H:i', $timestamp) : '',
        'create_timestamp' => $timestamp,
        'detail_url'       => $detailUrl,
        'refund_status'    => (int)($row['refund_status'] ?? 0),

        'binfo'            => ['name' => $typeName, 'logo' => ''],
        'bid'              => 0,
        'extra_info'       => [],
        // ... 其他必需字段

}
```

## 六、常见问题

### Q1：如何修改订单卡片样式？

**A：** 编辑 `/uniapp/components/unified-order-card/unified-order-card.vue` 的 `<style>` 部分。

### Q2：如何修改订单类型标签颜色？

**A：** 编辑 `/uniapp/components/unified-order-card/unified-order-card.vue` 的 `tagColor` 计算属性。

### Q3：如何添加更多筛选条件（如时间范围）？

**A：** 
1. 在前端页面添加时间选择器组件
2. 在接口请求中添加时间参数
3. 在后端 `buildWhere()` 方法中处理时间参数

### Q4：如何禁用某个订单类型？

**A：** 
1. 在后端 `isModuleEnabled()` 方法中返回 `false`
2. 或在前端 `enabledTypes` 数组中移除该类型

### Q5：订单详情页路径错误怎么办？

**A：** 检查后端 `$detailRoutes` 配置，确保路径正确。

### Q6：如何自定义空数据提示？

**A：** 编辑 `/uniapp/components/nodata/nodata.vue` 组件。

### Q7：分页不正确怎么办？

**A：** 检查后端 `aggregateAllTypes()` 方法中的 `fetchLimit` 计算逻辑。

### Q8：某个订单类型不显示怎么办？

**A：** 
1. 检查后端 `$orderTypes` 是否配置
2. 检查 `isModuleEnabled()` 是否返回 `true`
3. 检查数据表名是否正确

## 七、性能优化建议

### 1. 启用单表查询

当用户选择特定订单类型时，系统会自动使用单表查询（直通模式），性能等同于原有接口。

### 2. 缓存订单数量

在高并发场景下，可以将订单数量统计结果缓存到 Redis：

```php
public function ordercount()
{
    $cacheKey = 'order_count_' . aid . '_' . mid;
    $cached = cache($cacheKey);
    
    if ($cached) {
        return $this->json($cached);
    }
    
    $counts = [/* 统计逻辑 */];
    
    // 缓存5分钟
    cache($cacheKey, $counts, 300);
    
    return $this->json($counts);
}
```

### 3. 数据库索引

确保各订单表有以下索引：

```sql
-- 商城订单
ALTER TABLE shop_order ADD INDEX idx_aid_mid_status (aid, mid, status, createtime);

-- 拼团订单
ALTER TABLE collage_order ADD INDEX idx_aid_mid_status (aid, mid, status, createtime);

-- 选片订单
ALTER TABLE ai_travel_photo_order ADD INDEX idx_aid_uid_status (aid, uid, status, create_time);
```

### 4. 前端虚拟列表

如果订单数量非常多（100+），建议使用虚拟列表减少渲染压力：

```vue
<!-- 使用 uview-ui 的虚拟列表组件 -->
<u-list 
    :list="orderList" 
    @scrolltolower="loadMore">
    <template v-slot:default="{item}">
        <unified-order-card :item="item"></unified-order-card>
    </template>
</u-list>
```
