# 统一订单管理功能测试用例

## 测试环境准备

1. 确保后端 API 已部署
2. 确保 UniApp 前端代码已编译
3. 准备测试账号，该账号需要有不同类型的订单数据

## 测试用例

### 用例1：进入统一订单列表页

**测试步骤：**
1. 登录应用
2. 进入会员中心（个人中心）
3. 点击"查看全部订单"按钮

**预期结果：**
- 成功进入统一订单列表页
- 页面标题显示"我的订单"
- 显示订单列表（如果有订单数据）

---

### 用例2：状态筛选功能

**测试步骤：**
1. 在统一订单列表页
2. 点击顶部Tab："待付款"

**预期结果：**
- 只显示状态为"待付款"的订单
- Tab高亮显示

**重复测试：**
- 点击"待发货"、"待收货"、"已完成"、"退款/售后"
- 验证每个状态筛选都正确

---

### 用例3：订单类型筛选功能

**测试步骤：**
1. 在统一订单列表页
2. 横向滑动订单类型筛选栏
3. 点击"拼团"

**预期结果：**
- 只显示拼团订单
- "拼团"选项高亮显示（底部红线）
- 订单卡片顶部显示"拼团"标签

**重复测试：**
- 测试所有订单类型筛选
- 验证每个类型筛选都正确

---

### 用例4：关键字搜索功能

**测试步骤：**
1. 在统一订单列表页
2. 在搜索框输入已知订单号的前几位
3. 按回车确认搜索

**预期结果：**
- 显示匹配的订单
- 其他不匹配的订单被过滤

**重复测试：**
- 搜索商品名关键字
- 搜索完整订单号
- 搜索不存在的关键字（应显示空数据）

---

### 用例5：订单卡片展示

**测试步骤：**
1. 在统一订单列表页
2. 观察订单卡片

**预期结果：**
- 顶部栏：显示订单类型标签（彩色徽章）、订单号、订单状态
- 内容区：显示商品封面图、商品名称、数量描述
- 底部栏：显示"共计X件商品 实付:￥XX.XX"
- 操作区：显示"详情"按钮

**验证要点：**
- 不同订单类型的标签颜色不同
- 商品图片正常显示
- 金额格式正确

---

**重复测试：**

## 性能测试

### 测试1：大量订单加载性能

**测试步骤：**
1. 使用有100+订单的测试账号
2. 进入统一订单列表页
3. 观察首屏加载时间

**预期结果：**
- 首屏加载时间 < 2秒
- 滚动流畅，无明显卡顿

---

### 测试2：类型筛选性能

**测试步骤：**
1. 选择"全部"类型（聚合查询）
2. 记录加载时间
3. 选择"商城"类型（单表查询）
4. 记录加载时间

**预期结果：**
- 单表查询明显快于聚合查询
- 聚合查询加载时间 < 3秒

## API测试

### 测试1：统一订单列表接口

**请求示例：**
```
POST /api.php/ApiUnifiedOrder/orderlist
{
  "st": "all",
  "order_type": "all",
  "keyword": "",
  "pagenum": 1
}
```

**验证要点：**
- 返回数据结构正确
- datalist 数组包含订单数据
- 每个订单包含所有必需字段

---

### 测试2：按状态筛选接口

**请求示例：**
```
POST /api.php/ApiUnifiedOrder/orderlist
{
  "st": "0",
  "order_type": "all",
  "keyword": "",
  "pagenum": 1
}
```

**验证要点：**
- 返回的订单状态都是0（待付款）

---

### 测试3：按类型筛选接口

**请求示例：**
```
POST /api.php/ApiUnifiedOrder/orderlist
{
  "st": "all",
  "order_type": "shop",
  "keyword": "",
  "pagenum": 1
}
```

**验证要点：**
- 返回的订单类型都是 shop

---

### 测试4：关键字搜索接口

**请求示例：**
```
POST /api.php/ApiUnifiedOrder/orderlist
{
  "st": "all",
  "order_type": "all",
  "keyword": "202603270001",
  "pagenum": 1
}
```

**验证要点：**
- 返回的订单号包含关键字

---

### 测试5：订单数量统计接口

**请求示例：**
```
POST /api.php/ApiUnifiedOrder/ordercount
```

**验证要点：**
- 返回各状态订单数量
- 数量与实际订单数匹配
