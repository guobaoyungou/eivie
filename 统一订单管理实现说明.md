# UniApp 统一订单管理功能实现说明

## 一、功能概述

根据《订单统一管理.md》文档的要求，已完成会员中心"我的订单"统一管理功能。该功能将平台所有订单类型（商城、拼团、秒杀、团购、砍价、幸运拼团、积分兑换、预约、课程、周期购、选片等）统一聚合到一个页面中展示，提供一站式订单查看体验。

## 二、已完成的功能模块

### 2.1 后端实现

#### 文件位置
`/home/www/ai.eivie.cn/app/controller/ApiUnifiedOrder.php`

#### 核心功能

1. **统一订单列表接口** - `orderlist()`
   - 支持全部订单类型聚合查询
   - 支持按订单类型筛选（单表直通查询）
   - 支持按状态筛选（待付款、待发货、待收货、已完成、退款/售后）
   - 支持关键字搜索（订单号、商品名）
   - 应用层分页，每页10条

2. **统一订单数量统计接口** - `ordercount()`
   - 统计各状态订单总数
   - 返回待付款、待发货、待收货、已完成、退款/售后的数量

3. **统一订单详情接口** - `detail()`
   - 兼容选片订单的详情展示
   - 自动识别订单类型并调用对应的详情逻辑

#### 支持的订单类型

| 订单类型 | 标识 | 数据表 | 详情页路径 |
|---------|------|--------|-----------|
| 商城订单 | shop | shop_order | /pagesExt/order/detail |
| 拼团订单 | collage | collage_order | /activity/collage/orderdetail |
| 秒杀订单 | seckill | seckill_order | /activity/seckill/orderdetail |
| 团购订单 | tuangou | tuangou_order | /activity/tuangou/orderdetail |
| 砍价订单 | kanjia | kanjia_order | /activity/kanjia/orderdetail |
| 幸运拼团 | lucky_collage | lucky_collage_order | /activity/luckycollage/orderdetail |
| 积分兑换 | scoreshop | scoreshop_order | /activity/scoreshop/orderdetail |
| 预约订单 | yuyue | yuyue_order | /activity/yuyue/orderdetail |
| 课程订单 | kecheng | kecheng_order | /activity/kecheng/product |

#### 数据标准化

每个订单类型都被标准化为统一的数据格式：

```php
[
    'id'               => 订单ID,
    'order_type'       => 订单类型标识,
    'order_type_name'  => 订单类型中文名,
    'ordernum'         => 订单编号,
    'title'            => 订单标题,
    'cover_image'      => 封面图,
    'total_price'      => 实付金额,
    'status'           => 状态码,
    'status_text'      => 状态文本,
    'item_count'       => 商品件数,
    'create_time'      => 创建时间,
    'create_timestamp' => 时间戳,
    'detail_url'       => 详情页路径,
    'refund_status'    => 退款状态,
    'prolist'          => 商品列表,
    'extra_info'       => 特有信息
]
```

#### 状态码映射

统一状态码：
- 0 = 待付款
- 1 = 待发货
- 2 = 待收货
- 3 = 已完成
- 4 = 已关闭
- 10 = 退款/售后

特殊处理：
- 选片订单：status=0(待支付)→0(待付款)，status=1(待服务)→2(待收货)
- 课程订单：status=1(已支付)→3(已完成)

#### 性能优化

1. **类型筛选直通**：选择特定订单类型时，仅查询对应单表
2. **模块开关检测**：未开通的模块不参与查询
3. **应用层聚合**：避免数据库层 UNION，提高查询效率
4. **预取策略**：多表查询时，每表预取足够数据确保分页正确

## 四、功能特点

### 4.1 非侵入式设计

- ✅ 保留各模块原有的独立订单页面和接口不变
- ✅ 新增统一聚合层，不影响现有业务
- ✅ 各模块订单详情页保持原有路径

### 4.2 渐进式扩展

- ✅ 优先聚合核心订单类型
- ✅ 支持通过配置灵活增删订单类型
- ✅ 未开通的模块自动不参与查询

- ✅ 类型筛选时直通单表查询，性能等同原有接口

- ✅ 模块开关检测，减少无效查询
- ✅ 预取策略确保多表聚合时分页准确性

### 4.4 用户体验

- ✅ 统一的订单卡片样式，清晰易读
- ✅ 订单类型标签彩色区分，一目了然
- ✅ 支持多维度筛选（状态 + 类型 + 关键字）
- ✅ 流畅的下拉刷新和上拉加载
- ✅ 点击订单自动跳转对应详情页

## 五、使用指南

### 5.1 从会员中心进入

1. 进入会员中心（个人中心）
2. 点击"查看全部订单"按钮
3. 进入统一订单列表页

### 5.2 筛选订单

1. **按状态筛选**：点击顶部Tab（全部、待付款、待发货、待收货、已完成、退款/售后）
2. **按类型筛选**：点击类型筛选栏（全部、商城、拼团、秒杀、团购、砍价、选片等）
3. **关键字搜索**：在搜索框输入订单号或商品名，按回车搜索

### 5.3 查看订单详情

点击任意订单卡片或"详情"按钮，自动跳转到对应类型的详情页：
- 商城订单 → 商城订单详情页
- 拼团订单 → 拼团订单详情页
- 秒杀订单 → 秒杀订单详情页
- 选片订单 → 选片订单详情页
- 其他类型 → 对应详情页

## 六、技术亮点

### 6.1 统一数据格式

所有订单类型都被标准化为统一的数据结构，前端无需关心订单来源：

```javascript
// 前端接收到的统一格式
{
  id: 订单ID,
  order_type: 'shop|collage|seckill|...',
  order_type_name: '商城|拼团|秒杀|...',
  ordernum: 订单号,
  title: 订单标题,
  cover_image: 封面图,
  total_price: 实付金额,
  status: 统一状态码,
  status_text: 状态文本,
  item_count: 商品件数,
  create_time: 创建时间,
  detail_url: 详情页路径,
  prolist: 商品列表
}
```

### 6.2 智能路由分发

订单卡片组件根据 `order_type` 自动跳转到对应的详情页，无需硬编码：

```javascript
// 详情页路由映射（后端维护）
protected $detailRoutes = [
    'shop'          => '/pagesExt/order/detail?id=',
    'collage'       => '/activity/collage/orderdetail?id=',
    'seckill'       => '/activity/seckill/orderdetail?id=',
    // ...
];

// 前端接收到的 detail_url 已经拼接完成
item.detail_url = '/pagesExt/order/detail?id=1001'
```

### 6.3 状态码统一映射

不同订单类型的状态码被映射为统一状态码，前端筛选逻辑简化：

```php
// 选片订单状态映射
ai_pick: status=0(待支付) → unified=0(待付款)
ai_pick: status=1(待服务) → unified=2(待收货)

// 课程订单状态映射
kecheng: status=1(已支付) → unified=3(已完成)

// 其他订单直接映射
shop/collage/seckill/...: status=N → unified=N
```

### 6.4 适配器模式

每种订单类型有对应的标准化方法，易于扩展新类型：

```php
// 商城订单标准化
protected function normalizeShopOrder($row, $type, $typeName, $timestamp)

// 活动订单标准化
protected function normalizeActivityOrder($row, $type, $typeName, $timestamp)

// 选片订单标准化
protected function normalizeAiPickOrder($row, $type, $typeName, $timestamp)

// 课程订单标准化
protected function normalizeKechengOrder($row, $type, $typeName, $timestamp)
```
