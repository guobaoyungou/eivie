## 架构特点

### MVVM模式实现
- **完全分离**: View与ViewModel通过DataBinding通信，无直接依赖
- **命令模式**: 所有用户操作通过ICommand实现
- **属性通知**: 使用INotifyPropertyChanged自动同步UI
- **服务注入**: ViewModel通过构造函数注入Service依赖

### 事件驱动架构
- Service层通过事件通知ViewModel状态变化
- ViewModel订阅事件并更新属性
- UI通过属性绑定自动刷新
- 使用Dispatcher确保UI线程安全

### 错误处理策略
- 统一的异常捕获机制（SafeExecute/SafeExecuteAsync）
- 错误信息通过ErrorMessage属性暴露给UI
- 日志服务记录所有错误详情
- 用户友好的错误提示（MessageBox）

### 性能优化
- 日志显示限制最大1000条
- 使用虚拟化列表（准备用于LogView）
- 异步操作防止UI冻结
- 定时器优化（每秒更新而非毫秒级）

#### 2. 依赖注入配置
建议使用Microsoft.Extensions.DependencyInjection:
```csharp
// App.xaml.cs
protected override void OnStartup(StartupEventArgs e)
{
    var services = new ServiceCollection();
    
    // 注册Services
    services.AddSingleton<ConfigService>();
    services.AddSingleton<LogService>();
    services.AddSingleton<ApiClient>();
    services.AddSingleton<DeviceService>();
    services.AddSingleton<FileWatcherService>();
    services.AddSingleton<UploadService>();
    services.AddSingleton<HeartbeatService>();
    
    // 注册ViewModels
    services.AddSingleton<MainViewModel>();
    services.AddTransient<HomeViewModel>();
    services.AddTransient<SettingsViewModel>();
    services.AddTransient<LogViewModel>();
    services.AddTransient<AboutViewModel>();
    
    var serviceProvider = services.BuildServiceProvider();
    var mainWindow = new MainWindow
    {
        DataContext = serviceProvider.GetService<MainViewModel>()
    };
    mainWindow.Show();
}
```
