# AI旅拍商家Windows客户端开发文档

## 一、架构设计

### 1.1 总体架构

本客户端采用分层架构设计:

```
┌─────────────────────────────────────┐
│        展示层 (Presentation)         │
│    WPF界面 + ViewModels (MVVM)      │
├─────────────────────────────────────┤
│         业务层 (Business)            │
│  Services (设备/上传/监控/日志等)    │
├─────────────────────────────────────┤
│         数据层 (Data)                │
│  Models + Utils + Configuration     │
├─────────────────────────────────────┤
│       基础设施层 (Infrastructure)     │
│  .NET Framework + WPF + HttpClient  │
└─────────────────────────────────────┘
```

### 1.2 设计模式

**MVVM模式:**
- Model: 数据模型层
- View: XAML界面
- ViewModel: 视图模型,连接View和Model

**单例模式:**
- ConfigService: 配置管理服务
- LogService: 日志服务

**观察者模式:**
- 日志事件订阅
- 文件监控事件

**工厂模式:**
- 服务实例创建

## 三、数据流程

### 3.1 设备注册流程

```
客户端启动
    ↓
检查config.json
    ↓
是否已注册? ─No→ 显示注册向导
    │            ↓
    │       用户输入信息
    │            ↓
    │       获取硬件信息
    │            ↓
    │       发送注册请求
    │            ↓
    │       服务器验证
    │            ↓
    │       返回Token
    │            ↓
    │       保存配置
    ↓            ↓
Yes ←───────────┘
    ↓
进入主界面
```

### 3.2 文件监控上传流程

```
开始监控
    ↓
FileSystemWatcher监听 + 定时轮询
    ↓
检测到新文件
    ↓
文件过滤检查(类型/大小/临时文件)
    ↓
等待文件稳定(2秒)
    ↓
计算文件MD5
    ↓
检查是否重复
    ↓
添加到上传队列
    ↓
从队列取出任务(并发控制)
    ↓
上传到服务器
    ↓
上传成功? ─No→ 重试次数<5? ─Yes→ 延迟后重试
    │                 │
   Yes               No
    │                 │
    ↓                 ↓
更新统计        标记最终失败
    ↓                 ↓
从队列移除      记录错误日志
```

### 3.3 心跳保活流程

```
客户端启动
    ↓
启动心跳定时器(60秒间隔)
    ↓
定时触发
    ↓
构造心跳数据
    ↓
发送心跳请求
    ↓
服务器响应? ─No→ 失败次数+1 → 失败次数≥3? ─Yes→ 报警
    │                              │
   Yes                            No
    │                              │
    ↓                              ↓
更新最后心跳时间            等待下次心跳
    ↓
继续监听
```

## 四、关键技术实现

### 4.1 文件监控实现

使用混合模式(FileSystemWatcher + 定时轮询):

```csharp
// FileSystemWatcher实时监听
var watcher = new FileSystemWatcher(path);
watcher.Created += OnFileCreated;
watcher.EnableRaisingEvents = true;

// 定时轮询作为补充
var timer = new Timer(ScanDirectory, null, 0, scanInterval * 1000);

// 文件稳定性检测
private bool IsFileStable(string filePath)
{
    long size1 = new FileInfo(filePath).Length;
    Thread.Sleep(2000); // 等待2秒
    long size2 = new FileInfo(filePath).Length;
    return size1 == size2 && size1 > 0;
}
```

### 4.2 并发上传控制

使用SemaphoreSlim实现并发控制:

```csharp
private SemaphoreSlim _semaphore;

public void StartUpload()
{
    _semaphore = new SemaphoreSlim(concurrentCount, concurrentCount);
    
    // 启动上传线程池
    for (int i = 0; i < concurrentCount; i++)
    {
        Task.Run(() => UploadWorker());
    }
}

private async Task UploadWorker()
{
    while (_running)
    {
        await _semaphore.WaitAsync();
        try
        {
            var task = GetNextTask();
            if (task != null)
            {
                await UploadFile(task);
            }
        }
        finally
        {
            _semaphore.Release();
        }
    }
}
```

### 4.3 失败重试机制

使用指数退避策略:

```csharp
private int[] _retryDelays = { 5, 10, 20, 40, 60 }; // 秒

private async Task<bool> UploadWithRetry(UploadTask task)
{
    for (int i = 0; i < maxRetry; i++)
    {
        try
        {
            var result = await _apiClient.UploadFileAsync(task.FilePath, task.Md5, task.FileSize);
            if (result.Code == 200)
            {
                return true;
            }
        }
        catch (Exception ex)
        {
            _logger.Error("UploadService", $"上传失败: {ex.Message}");
        }
        
        task.RetryCount++;
        if (task.RetryCount < maxRetry)
        {
            int delay = _retryDelays[Math.Min(task.RetryCount, _retryDelays.Length - 1)];
            await Task.Delay(delay * 1000);
        }
    }
    
    return false;
}
```

### 4.4 配置加密存储

敏感信息使用AES加密:

```csharp
public bool SaveConfig(ConfigModel config)
{
    // 克隆配置对象
    var configToSave = JsonConvert.DeserializeObject<ConfigModel>(
```

## 五、开发规范

### 5.1 命名规范

- 类名: 大驼峰(PascalCase), 如: `DeviceService`
- 方法名: 大驼峰(PascalCase), 如: `RegisterDevice()`
- 私有字段: 下划线+小驼峰, 如: `_httpClient`
- 参数/局部变量: 小驼峰(camelCase), 如: `deviceId`
- 常量: 全大写+下划线, 如: `MAX_RETRY_COUNT`

### 5.2 注释规范

所有公共类和方法必须添加XML文档注释:

```csharp
/// <summary>
/// 上传文件到服务器
/// </summary>
/// <param name="filePath">文件路径</param>
/// <param name="md5">文件MD5值</param>
/// <param name="fileSize">文件大小</param>
/// <returns>上传结果</returns>
public async Task<FileUploadResponse> UploadFileAsync(
    string filePath, string md5, long fileSize)
{
    // 实现代码
}
```

### 5.3 异常处理

- 使用try-catch捕获可能的异常
- 记录详细的错误日志
- 向用户显示友好的错误提示
- 不要吞掉异常

```csharp
try
{
    // 业务逻辑
}
catch (Exception ex)
{
    _logger.Error("ModuleName", $"操作失败: {ex.Message}", ex);
    MessageBox.Show("操作失败,请查看日志获取详细信息");
}
```

### 5.4 日志规范

- 关键操作必须记录日志
- 使用合适的日志级别
- 日志信息要详细明确

```csharp
_logger.Info("UploadService", $"开始上传文件: {fileName}");
_logger.Warn("FileWatcher", $"文件过大被跳过: {filePath}");
_logger.Error("ApiClient", $"API请求失败: {ex.Message}", ex);
```
