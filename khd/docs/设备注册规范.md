## 1. 概述

### 1.1 什么是设备注册

设备注册是AI旅拍商家客户端首次运行时，向服务器申请设备认证凭证的过程。注册成功后，客户端将获得：

- **设备ID（device_id）**：设备的唯一标识符
- **设备令牌（device_token）**：用于API认证的密钥

### 1.2 注册场景

| 场景 | 说明 | 处理方式 |
|------|------|---------|
| **首次安装** | 客户端首次运行，本地无配置 | 引导用户完成注册 |
| **重新安装** | 客户端卸载后重装 | 使用原令牌或重新注册 |
| **令牌失效** | 令牌被禁用或删除 | 重新注册获取新令牌 |
| **迁移设备** | 在新计算机上运行 | 作为新设备注册 |

### 1.3 前置条件

在进行设备注册前，需要确保：

✅ 商家已在后台生成设备令牌  
✅ 客户端能够访问API服务器  
✅ 客户端能够获取系统硬件信息  
✅ 商家ID（bid）和门店ID（mdid）已知

```csharp
public class DeviceInfo
{
    public string GetMacAddress()
    {
        // 获取第一个有效网络适配器的MAC地址
        foreach (NetworkInterface nic in NetworkInterface.GetAllNetworkInterfaces())
        {
            if (nic.OperationalStatus == OperationalStatus.Up)
            {
                return nic.GetPhysicalAddress().ToString();
            }
        }
        return "UNKNOWN";
    }

    public string GetComputerName()
    {
        return Environment.MachineName;
    }

    public string GetOSVersion()
    {
        return Environment.OSVersion.ToString();
    }

    public string GetLocalIPAddress()
    {
        var host = Dns.GetHostEntry(Dns.GetHostName());
        foreach (var ip in host.AddressList)
        {
            if (ip.AddressFamily == AddressFamily.InterNetwork)
            {
                return ip.ToString();
            }
        }
        return "127.0.0.1";
    }
}
```

## 4. 设备信息采集

### 4.1 MAC地址获取

**C# 代码示例：**
```csharp
using System.Net.NetworkInformation;

public string GetMacAddress()
{
    try
    {
        foreach (NetworkInterface nic in NetworkInterface.GetAllNetworkInterfaces())
        {
            // 排除虚拟网卡和未激活的网卡
            if (nic.OperationalStatus == OperationalStatus.Up &&
                nic.NetworkInterfaceType != NetworkInterfaceType.Loopback &&
                nic.NetworkInterfaceType != NetworkInterfaceType.Tunnel)
            {
                var mac = nic.GetPhysicalAddress().ToString();
                if (!string.IsNullOrEmpty(mac) && mac != "000000000000")
                {
                    // 格式化为 XX:XX:XX:XX:XX:XX
                    return string.Join(":", 
                        Enumerable.Range(0, mac.Length / 2)
                        .Select(i => mac.Substring(i * 2, 2)));
                }
            }
        }
    }
    catch (Exception ex)
    {
        Logger.Error("获取MAC地址失败", ex);
    }
    
    // 失败时生成随机MAC地址
    return GenerateRandomMac();
}

private string GenerateRandomMac()
{
    var random = new Random();
    var bytes = new byte[6];
    random.NextBytes(bytes);
    return string.Join(":", bytes.Select(b => b.ToString("X2")));
}
```

### 4.2 系统信息获取

**C# 代码示例：**
```csharp
using System.Management;

public class SystemInfo
{
    public string GetCpuInfo()
    {
        try
        {
            using (var searcher = new ManagementObjectSearcher("SELECT Name FROM Win32_Processor"))
            {
                foreach (var obj in searcher.Get())
                {
                    return obj["Name"]?.ToString() ?? "Unknown CPU";
                }
            }
        }
        catch { }
        return "Unknown CPU";
    }

    public string GetMemorySize()
    {
        try
        {
            var totalMemory = new Microsoft.VisualBasic.Devices.ComputerInfo().TotalPhysicalMemory;
            return $"{totalMemory / 1024 / 1024 / 1024}GB";
        }
        catch { }
        return "Unknown";
    }

    public string GetDiskInfo()
    {
        try
        {
            var drives = DriveInfo.GetDrives()
                .Where(d => d.IsReady && d.DriveType == DriveType.Fixed)
                .Select(d => $"{d.Name} {d.TotalSize / 1024 / 1024 / 1024}GB");
            return string.Join(", ", drives);
        }
        catch { }
        return "Unknown";
    }
}
```

## 5. 令牌管理

### 5.1 令牌存储

**推荐存储方式：**

1. **配置文件（推荐）**
   - 位置：`%APPDATA%/AiTravelClient/config.json`
   - 优点：跨平台、易于备份
   - 缺点：可能被用户误删

2. **注册表**
   - 位置：`HKEY_CURRENT_USER\Software\AiTravelClient`
   - 优点：不易被误删
   - 缺点：Windows专用

3. **加密存储**
   - 使用 DPAPI 或 AES 加密
   - 防止令牌泄露

**代码示例：**
```csharp
using System.Security.Cryptography;
using System.Text;

public class TokenStorage
{
    private readonly string _configPath;

    public TokenStorage()
    {
        var appData = Environment.GetFolderPath(Environment.SpecialFolder.ApplicationData);
        var appFolder = Path.Combine(appData, "AiTravelClient");
        Directory.CreateDirectory(appFolder);
        _configPath = Path.Combine(appFolder, "config.json");
    }

    public void SaveToken(string deviceToken, string deviceId)
    {
        var config = new
        {
            device_id = deviceId,
            device_token = Encrypt(deviceToken),
            saved_at = DateTime.Now
        };
        
        var json = JsonConvert.SerializeObject(config, Formatting.Indented);
        File.WriteAllText(_configPath, json);
    }

    public string LoadToken()
    {
        if (!File.Exists(_configPath))
            return null;

        var json = File.ReadAllText(_configPath);
        var config = JsonConvert.DeserializeObject<dynamic>(json);
        return Decrypt(config.device_token.ToString());
    }

    private string Encrypt(string plainText)
    {
        var data = Encoding.UTF8.GetBytes(plainText);
        var encrypted = ProtectedData.Protect(data, null, DataProtectionScope.CurrentUser);
        return Convert.ToBase64String(encrypted);
    }

    private string Decrypt(string encryptedText)
    {
        var data = Convert.FromBase64String(encryptedText);
        var decrypted = ProtectedData.Unprotect(data, null, DataProtectionScope.CurrentUser);
        return Encoding.UTF8.GetString(decrypted);
    }
}
```

```csharp
public async Task<bool> RefreshTokenAsync()
{
    // 1. 清除本地令牌
    _tokenStorage.Clear();
    
    // 2. 显示注册界面

    var result = registerDialog.ShowDialog();
```

## 6. 错误处理

### 6.1 常见错误码

| 错误码 | 错误信息 | 原因 | 解决方案 |
|--------|----------|------|---------|
| 400 | 设备编码不能为空 | 未提供device_code | 检查参数 |
| 400 | 商家ID不能为空 | 未提供bid | 检查参数 |
| 401 | 设备Token无效 | 令牌错误或已失效 | 重新注册 |
| 403 | 设备已被禁用 | 设备被后台禁用 | 联系管理员 |
| 409 | 设备已存在 | device_code重复 | 使用已有令牌 |
| 500 | 数据库错误 | 服务器内部错误 | 联系技术支持 |

### 6.2 错误处理策略

**网络错误：**
```csharp
try
{
    var response = await _apiClient.RegisterDeviceAsync(deviceInfo);
    return response;
}
catch (HttpRequestException ex)
{
    // 网络连接失败
    ShowError("网络连接失败，请检查网络设置");
    Logger.Error("注册失败：网络错误", ex);
}
catch (TimeoutException ex)
{
    // 请求超时
    ShowError("请求超时，请稍后重试");
    Logger.Error("注册失败：超时", ex);
}
```

**业务错误：**
```csharp
if (response.Code != 200)
{
    switch (response.Code)
    {
        case 400:
            ShowError($"参数错误：{response.Msg}");
            break;
        case 401:
            ShowError("设备令牌无效，请检查令牌是否正确");
            break;
        case 403:
            ShowError("设备已被禁用，请联系管理员");
            break;
        case 409:
            ShowError("设备已存在，请使用已有令牌");
            break;
        default:
            ShowError($"注册失败：{response.Msg}");
            break;
    }
    return false;
}
```

### 6.3 重试机制

```csharp
public async Task<ApiResponse> RegisterWithRetryAsync(DeviceInfo deviceInfo, int maxRetries = 3)
{
    for (int i = 0; i < maxRetries; i++)
    {
        try
        {
            var response = await _apiClient.RegisterDeviceAsync(deviceInfo);
            if (response.Code == 200)
            {
                return response;
            }
            
            // 业务错误不重试
            if (response.Code >= 400 && response.Code < 500)
            {
                return response;
            }
        }
        catch (Exception ex)
        {
            Logger.Warning($"注册失败，第{i + 1}次重试", ex);
            
            if (i < maxRetries - 1)
            {
                // 指数退避：2秒、4秒、8秒
                await Task.Delay(TimeSpan.FromSeconds(Math.Pow(2, i + 1)));
            }
        }
    }
    
    throw new Exception("注册失败：已达到最大重试次数");
}
```

## 7. 安全规范

### 7.1 令牌安全

✅ **必须做：**
- 令牌加密存储（使用DPAPI或AES）
- HTTPS通信（避免中间人攻击）
- 令牌定期轮换（建议30-90天）
- 日志中不记录完整令牌

❌ **禁止做：**
- 明文存储令牌
- HTTP通信
- 在日志中输出令牌
- 在界面上长期显示令牌

### 7.2 数据传输安全

## 9. 常见问题

### Q1: 如何获取设备令牌？

**A:** 登录商家后台 → 旅拍 → 设备管理 → 点击"生成新设备令牌"按钮，填写设备名称和选择门店后生成。

### Q2: 设备令牌丢失怎么办？

**A:** 有两种方式：
1. 在后台设备管理页面查看已生成的令牌
2. 重新生成新令牌（旧令牌仍然有效）

### Q3: 一个设备可以绑定多个门店吗？

**A:** 不可以。一个设备只能绑定一个门店（mdid）。如需更换门店，需要重新注册。

### Q4: 客户端卸载后重装，需要重新注册吗？

**A:** 
- 如果配置文件仍存在：不需要，直接使用
- 如果配置文件已删除：需要重新输入令牌注册

### Q5: 设备令牌有效期是多久？

**A:** 设备令牌永久有效，除非：
- 后台手动禁用设备
- 后台删除设备
- 商家账号被停用

### Q6: 如何处理"设备已存在"错误？

**A:** 说明该MAC地址已注册过。解决方案：
1. 查找之前的配置文件
2. 联系管理员重置设备
3. 使用不同的MAC地址（更换网卡）

### Q7: 注册时提示"商家ID不能为空"？

**A:** 检查后台生成令牌时是否正确选择了商家和门店。令牌中包含商家信息。

### Q8: 可以在多台电脑上使用同一个令牌吗？

**A:** 不建议。每台电脑应该有独立的令牌，便于管理和追踪。

### Q9: 如何测试注册是否成功？

**A:** 注册成功后：
1. 客户端能够进入主界面
2. 后台设备管理页面能看到设备在线
3. 心跳接口调用成功

### Q10: 令牌泄露了怎么办？

**A:** 立即在后台设备管理中禁用该设备，然后重新生成新令牌。
