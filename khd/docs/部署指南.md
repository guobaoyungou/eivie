# AI旅拍商家Windows客户端部署指南

## 一、部署前准备

### 1.1 系统要求检查

在部署前,请确保目标计算机满足以下要求:

**最低配置:**
- 操作系统: Windows 7 SP1 64位或更高版本
- .NET Framework: 4.7.2或更高版本
- CPU: 双核 2.0GHz
- 内存: 2GB
- 磁盘空间: 100MB
- 网络: 稳定的宽带连接

**推荐配置:**
- 操作系统: Windows 10/11 64位
- .NET Framework: 4.8
- CPU: 四核 2.5GHz及以上
- 内存: 4GB及以上
- 磁盘空间: 500MB
- 网络: 有线网络连接

### 1.2 网络环境检查

- 确保计算机可以访问服务器API地址
- 检查防火墙设置,允许客户端访问互联网
- 建议使用有线网络以保证稳定性

### 1.3 权限要求

- 需要管理员权限进行安装
- 需要读写权限访问监控文件夹
- 需要网络访问权限

## 二、安装步骤

### 2.1 安装.NET Framework

如果系统未安装.NET Framework 4.7.2或更高版本:

1. 下载.NET Framework 4.7.2安装包
2. 以管理员身份运行安装程序
3. 按照向导完成安装
4. 重启计算机

### 2.2 安装客户端程序

**方式一: 使用安装程序(推荐)**

1. 下载 `AiTravelClient_Setup_v1.0.0.exe`
2. 右键点击安装程序,选择"以管理员身份运行"
3. 选择安装路径(默认: C:\Program Files\AiTravelClient)
4. 选择是否创建桌面快捷方式
5. 点击"安装"开始安装
6. 安装完成后点击"完成"

**方式二: 手动部署**

1. 解压客户端压缩包到目标目录
2. 确保目录结构完整:
   ```
   AiTravelClient/
   ├── AiTravelClient.exe
   ├── config.json
   ├── Newtonsoft.Json.dll
   ├── System.Management.dll
   └── logs/
   ```
3. 创建桌面快捷方式(可选)

## 三、首次配置

### 3.1 获取设备编码

在使用客户端前,需要先在后台管理系统中创建设备记录:

1. 登录商家后台管理系统
2. 进入"AI旅拍" → "设备管理"
3. 点击"添加设备"
4. 填写设备信息:
   - 设备名称: 例如"摄影门店A号机"
   - 门店: 选择所属门店
   - 备注: 可选
5. 保存后获得设备编码,记录备用

### 3.2 启动客户端

1. 双击桌面快捷方式或程序目录中的 `AiTravelClient.exe`
2. 首次启动会显示设备注册向导

### 3.3 设备注册

在注册向导中填写以下信息:

1. **服务器地址**: 
   - 输入API服务器地址,例如: `https://your-domain.com`
   - 点击"测试连接"验证地址是否正确

2. **设备信息**:
   - 设备编码: 输入从后台获取的设备编码
   - 商家ID: 输入商家ID(从后台查询)
   - 门店ID: 输入门店ID(可选)
   - 设备名称: 自定义设备名称

3. **点击"注册设备"**:
   - 系统会自动获取计算机硬件信息
   - 向服务器发送注册请求
   - 成功后会保存设备Token到配置文件

4. **注册成功提示**:
   - 显示"注册成功"消息
   - 自动进入主界面

### 3.4 配置监控路径

在主界面中配置需要监控的文件夹:

1. 点击"设置"选项卡
2. 在"监控设置"区域点击"添加文件夹"
3. 选择需要监控的文件夹路径
4. 可以添加多个监控路径
5. 点击"保存配置"

### 3.5 配置上传参数

根据实际情况调整上传参数:

1. **自动上传**: 开启/关闭自动上传
2. **并发上传数**: 1-10,默认3(根据网络情况调整)
3. **最大重试次数**: 默认5次
4. **上传超时**: 默认120秒

### 3.6 开始监控

1. 确认配置无误后
2. 点击主界面的"开始监控"按钮
3. 系统状态显示"监听中"
4. 新增的图片文件会自动被检测并上传

## 四、验证部署

### 4.1 功能测试

**测试文件检测:**
1. 复制一张JPG图片到监控文件夹
2. 观察客户端日志,应该显示"检测到新文件"
3. 文件应自动加入上传队列

**测试文件上传:**
1. 观察"上传统计"区域
2. "正在上传"数量应增加
3. 上传完成后"今日上传成功"数量应+1
4. 日志显示"上传成功"消息

**测试心跳:**
1. 客户端运行1分钟后
2. 查看日志,应该有"心跳发送成功"的记录
3. 在后台管理系统中查看设备在线状态

### 4.2 异常处理测试

**网络中断测试:**
1. 断开网络连接
2. 添加图片到监控文件夹
3. 客户端应显示"网络异常"
4. 恢复网络后,文件应自动重试上传

**重复文件测试:**
1. 上传一张图片
2. 再次复制相同的图片(MD5相同)
3. 服务器应返回"重复文件"信息
4. 不会产生重复记录

## 五、运维管理

### 5.1 日志查看

**查看实时日志:**
- 在客户端主界面底部可以看到实时滚动的日志

**查看历史日志:**
1. 打开程序安装目录
2. 进入 `logs/` 文件夹
3. 按日期查找日志文件:
   - `runtime_YYYYMMDD.log`: 运行日志
   - `error_YYYYMMDD.log`: 错误日志

**导出日志:**
1. 点击"日志"选项卡
2. 选择日期范围和日志级别
3. 点击"导出"按钮
4. 选择保存位置

### 5.2 配置修改

**修改服务器地址:**
1. 停止监控
2. 进入"设置"选项卡
3. 修改服务器地址
4. 点击"测试连接"验证
5. 保存配置

**修改监控路径:**
1. 进入"设置" → "监控设置"
2. 删除不需要的路径
3. 添加新的监控路径
4. 保存配置

**调整性能参数:**
- 根据网络带宽调整并发上传数
- 根据文件大小调整超时时间
- 根据业务需求调整扫描间隔

### 5.3 故障排查

**问题: 客户端无法启动**
- 检查.NET Framework是否安装
- 检查config.json文件是否存在
- 以管理员身份运行
- 查看Windows事件日志

**问题: 文件无法上传**
- 检查网络连接
- 检查设备Token是否有效
- 检查服务器地址是否正确
- 查看error日志文件

**问题: 文件未被检测到**
- 检查文件类型是否为JPG/PNG
- 检查文件大小是否在10KB-10MB范围内
- 检查监控路径是否正确
- 检查是否为临时文件

**问题: 上传速度慢**
- 检查网络带宽
- 增加并发上传数
- 使用有线网络
- 检查服务器性能

### 5.4 性能优化

**优化建议:**
1. 使用有线网络而非WiFi
2. 根据带宽调整并发数(10Mbps建议2-3个并发)
3. 定期清理日志文件
4. 避免监控包含大量文件的目录
5. 关闭不必要的后台程序

**监控指标:**
- CPU占用率应 < 10%
- 内存占用应 < 300MB
- 网络上传带宽使用合理
- 上传成功率应 > 95%

### 5.5 备份与恢复

**备份配置:**
1. 复制 `config.json` 文件
2. 记录设备Token(已加密)
3. 保存到安全位置

**恢复配置:**
1. 将备份的 `config.json` 复制回程序目录
2. 重启客户端
3. 验证设备连接状态

## 六、升级指南

### 6.1 版本升级

**升级前准备:**
1. 备份 `config.json` 配置文件
2. 导出重要日志
3. 记录当前配置参数

**升级步骤:**
1. 关闭正在运行的客户端
2. 下载新版本安装包
3. 运行安装程序(选择覆盖安装)
4. 验证配置文件是否保留
5. 启动新版本客户端
6. 检查功能是否正常

**升级回滚:**
- 如果新版本有问题,卸载后安装旧版本
- 恢复备份的配置文件

### 6.2 自动升级(待实现)

未来版本将支持:
- 启动时自动检测新版本

## 七、安全建议

### 7.1 网络安全

- 使用HTTPS协议与服务器通信
- 定期更新设备Token
- 不要在公共网络环境下使用
- 配置防火墙规则

### 7.2 数据安全

- Token已加密存储在配置文件中
- 定期备份配置文件
- 限制对config.json的访问权限
- 日志中不记录敏感信息

### 7.3 访问控制

- 限制只有授权人员访问客户端
- 设置Windows用户密码
- 不要在无人看管时保持登录

## 八、卸载指南

### 8.1 卸载步骤

1. 打开Windows控制面板
2. 进入"程序和功能"
3. 找到"AI旅拍商家客户端"
4. 右键点击,选择"卸载"
5. 选择是否保留配置和日志
   - 保留: 仅删除程序文件
   - 完全删除: 删除所有文件和配置

### 8.2 手动清理

如需手动清理残留文件:

1. 删除程序安装目录
2. 删除桌面快捷方式
3. 删除开始菜单项
4. (可选)删除配置和日志文件
