# 商城订单列表页接口恢复说明

## 问题描述

访问 `https://ai.eivie.cn/h5/1.html#/pagesExt/order/orderlist?st=0` 时页面显示空白。

## 问题原因

原商城订单列表页 (`/pagesExt/order/orderlist.vue`) 之前被修改为调用统一订单接口 `ApiUnifiedOrder/orderlist`，但没有指定 `order_type` 参数，导致查询了所有类型的订单而不是只查询商城订单。

由于统一订单接口的数据格式与原商城订单接口略有不同，前端需要进行复杂的数据转换，这增加了复杂度和出错的可能性。

## 解决方案

将商城订单列表页恢复为使用原来的商城订单接口 `ApiOrder/orderlist`，符合以下设计原则：

1. **非侵入式设计**：各模块原有的独立订单页面和接口保持不变
2. **职责分离**：
   - 商城订单列表页 (`/pagesExt/order/orderlist`) → 使用商城订单接口 (`ApiOrder/orderlist`)
   - 统一订单列表页 (`/pagesExt/order/unifiedOrderlist`) → 使用统一订单接口 (`ApiUnifiedOrder/orderlist`)

## 修改内容

### 文件：`/uniapp/pagesExt/order/orderlist.vue`

#### 修改1：恢复原商城订单接口调用

**修改前：**
```javascript
// 使用统一订单接口
app.post('ApiUnifiedOrder/orderlist', {st: st,pagenum: pagenum,keyword:that.keyword}, function (res) {
    that.loading = false;
    var data = res.datalist;
    // 将统一订单格式转换为商城订单格式
    var transformedData = [];
    if(data && data.length > 0){
      transformedData = that.transformOrderData(data);
    }
    if (pagenum == 1) {
        that.datalist = transformedData;
        if (transformedData.length == 0) {
            that.nodata = true;
        }
        that.loaded();
    }else{
        if (transformedData.length == 0) {
            that.nomore = true;
        } else {
            var datalist = that.datalist;
            var newdata = datalist.concat(transformedData);
            that.datalist = newdata;
        }
    }
});
```

**修改后：**
```javascript
// 使用原商城订单接口
app.post('ApiOrder/orderlist', {st: st,pagenum: pagenum,keyword:that.keyword}, function (res) {
    that.loading = false;
    var data = res.datalist;
    if (pagenum == 1) {
        that.datalist = data;
        if (data.length == 0) {
            that.nodata = true;
        }
        that.loaded();
    }else{
        if (data.length == 0) {
            that.nomore = true;
        } else {
            var datalist = that.datalist;
            var newdata = datalist.concat(data);
            that.datalist = newdata;
        }
    }
});
```

#### 修改2：删除不需要的数据转换方法

删除了 `transformOrderData` 方法（134行代码），因为直接使用商城订单接口返回的数据，不需要转换。

## 页面访问路径

### 商城订单列表页（原有页面）
- **路径**：`/pagesExt/order/orderlist`
- **接口**：`ApiOrder/orderlist`
- **用途**：查看商城订单（独立业务页面）
- **入口**：
  - 商品详情页的"查看订单"
  - 购买成功后的"查看订单"
  - 其他商城相关的订单入口

### 统一订单列表页（新增页面）
- **路径**：`/pagesExt/order/unifiedOrderlist`
- **接口**：`ApiUnifiedOrder/orderlist`
- **用途**：查看所有类型订单（会员中心统一入口）
- **入口**：
  - 会员中心的"查看全部订单"
  - 会员中心的状态快捷入口（待付款、待发货等）
  - 会员中心的积分兑换订单入口
