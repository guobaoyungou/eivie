# AI旅拍商家客户端 - 开发文档

## 开发环境配置

### 必备工具
- Node.js 16.x 或 18.x
- npm/yarn/pnpm
- Git
- VS Code（推荐）

### VS Code 推荐扩展
- Vue Language Features (Volar)
- TypeScript Vue Plugin (Volar)
- ESLint
- Prettier

## 项目结构说明

### 主进程（main/）
主进程负责应用的核心业务逻辑和系统级操作。

#### 核心文件
- `index.ts` - 应用入口，管理生命周期
- `window.ts` - 窗口创建与管理
- `tray.ts` - 系统托盘功能
- `ipc-handlers.ts` - IPC通信处理

#### 服务层（services/）
- `config.service.ts` - 配置管理（electron-store）
- `device.service.ts` - 设备注册与认证
- `file-watcher.service.ts` - 文件监控（chokidar）
- `upload.service.ts` - 文件上传管理
- `heartbeat.service.ts` - 心跳保活
- `api-client.service.ts` - HTTP客户端（axios）

#### 工具层（utils/）
- `crypto.util.ts` - 加密工具
- `file.util.ts` - 文件处理
- `system-info.util.ts` - 系统信息采集
- `retry.util.ts` - 重试策略

### 渲染进程（renderer/）
渲染进程负责用户界面展示和交互。

#### 页面视图（views/）
- `Home.vue` - 首页监控仪表板
- `Upload.vue` - 上传任务列表
- `Settings.vue` - 设置页面
- `Logs.vue` - 日志查看
- `About.vue` - 关于页面

#### 状态管理（stores/）
使用 Pinia 管理应用状态。

### 预加载脚本（preload/）
提供安全的IPC通信桥接，连接主进程和渲染进程。

## 开发规范

### 代码风格
- 使用 TypeScript 进行类型安全开发
- 遵循 ESLint 配置的代码规范
- 使用 Prettier 格式化代码

### 命名规范
- 文件：kebab-case（如：`config.service.ts`）
- 组件：PascalCase（如：`StatusCard.vue`）
- 变量/函数：camelCase（如：`getUserInfo`）
- 常量：UPPER_SNAKE_CASE（如：`MAX_RETRY_COUNT`）
- 类型/接口：PascalCase（如：`DeviceConfig`）

### Git 提交规范
```
<type>(<scope>): <subject>

<body>

<footer>
```

类型（type）：
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 代码格式调整
- refactor: 重构
- perf: 性能优化
- test: 测试相关
- chore: 构建/工具链相关

## 调试技巧

### 主进程调试
1. 查看终端输出的日志
2. 使用 `console.log` 或 `log.info`
3. 日志文件位于用户数据目录的 `logs/` 文件夹

### 渲染进程调试
1. 开发模式自动打开 Chrome DevTools
2. 使用 Vue DevTools 扩展
3. 使用 `console.log` 调试

### IPC 通信调试
在 preload 和 ipc-handlers 中添加日志输出。

## 常见问题

### 依赖安装问题
使用国内镜像：
```bash
npm config set registry https://registry.npmmirror.com
```

### Electron 下载慢
设置镜像：
```bash
export ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
```

### 热重载不工作
重启开发服务器。

## 性能优化建议

1. 避免在主进程执行耗时操作
2. 合理使用 IPC 通信，避免频繁调用
3. 大文件处理使用流式读取
4. 及时清理定时器和监听器

## 安全注意事项

1. 敏感信息加密存储
2. 使用 contextIsolation 和 preload
3. 禁用 nodeIntegration
4. 验证用户输入
5. 使用 HTTPS 通信
