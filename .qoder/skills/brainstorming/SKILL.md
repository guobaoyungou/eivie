# 设计细化（Brainstorming）

## 何时使用

此技能应该在以下情况下自动激活：
- 用户提出新的功能需求或想法
- 在开始编写任何代码之前
- 用户请求设计讨论或方案评估
- 需要澄清需求或探索实现方案时

## 核心原则

1. **先理解，再设计**：在提出解决方案之前，确保完全理解问题
2. **苏格拉底式提问**：通过问题引导用户明确需求
3. **探索替代方案**：考虑多种实现方式，比较优劣
4. **分段展示**：将设计分成易于理解的小块
5. **证据优先**：基于事实和最佳实践，而非猜测

## 工作流程

### 阶段1：需求澄清

通过提问明确以下内容：

**核心问题：**
- 这个功能要解决什么问题？
- 谁会使用这个功能？
- 期望的用户体验是什么？

**约束条件：**
- 技术约束（性能、兼容性、安全性）
- 时间约束（截止日期、优先级）
- 资源约束（团队规模、预算）

**成功标准：**
- 如何判断功能是否成功？
- 有哪些可衡量的指标？

### 阶段2：方案探索

提出2-3种不同的实现方案：

**方案A：[方案名称]**
- 描述：简要说明实现方式
- 优点：列出主要优势
- 缺点：列出潜在问题
- 适用场景：何时使用此方案

**方案B：[方案名称]**
- 描述：简要说明实现方式
- 优点：列出主要优势
- 缺点：列出潜在问题
- 适用场景：何时使用此方案

**方案C：[方案名称]**
- 描述：简要说明实现方式
- 优点：列出主要优势
- 缺点：列出潜在问题
- 适用场景：何时使用此方案

### 阶段3：设计细化

在用户选择方案后，详细展开设计：

**架构设计：**
- 整体架构图（用文字描述）
- 组件划分和职责
- 数据流
- 接口设计

**技术选型：**
- 使用的框架和库
- 为什么选择这些技术
- 替代方案考虑

**实现细节：**
- 关键算法或逻辑
- 数据模型
- API设计（如适用）

**风险评估：**
- 可能的技术风险
- 缓解措施
- 回退计划

### 阶段4：设计文档

将设计保存为结构化文档：

```markdown
# [功能名称] 设计文档

## 概述
[简要描述功能和目标]

## 需求分析
### 用户故事
作为 <用户类型>
我想要 <动作/目标>
以便 <收益/价值>

### 功能需求
- [ ] 需求1
- [ ] 需求2

### 非功能需求
- 性能要求
- 安全要求
- 可维护性要求

## 设计方案
### 架构设计
[详细架构说明]

### 技术选型
[技术栈说明]

### 数据模型
[数据结构设计]

### API设计
[接口定义]

## 实施计划
### 阶段划分
- 阶段1：[任务]
- 阶段2：[任务]
- 阶段3：[任务]

### 验收标准
- [ ] 标准1
- [ ] 标准2

## 风险评估
| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 风险1 | 高 | 中 | 措施1 |
| 风险2 | 中 | 低 | 措施2 |

## 参考资料
- [相关文档链接]
- [最佳实践参考]
```

## 最佳实践

1. **保持对话性**：不要一次性抛出所有信息，让用户参与讨论
2. **使用具体例子**：用实际场景说明抽象概念
3. **可视化设计**：尽可能使用图表或伪代码
4. **记录决策**：说明为什么选择某个方案而非其他
5. **保持开放**：随时准备根据反馈调整设计

## 常见问题

**Q: 用户需求不明确怎么办？**
A: 使用"5个为什么"技术深入挖掘真实需求

**Q: 如何在多个方案中做出选择？**
A: 创建决策矩阵，评估每个方案在关键维度上的表现

**Q: 设计文档应该多详细？**
A: 足够详细到让另一个开发者可以实施，但不要过度设计（YAGNI原则）

**Q: 如何处理技术债务？**
A: 在设计阶段就考虑可维护性，避免为了速度牺牲质量

## 输出检查清单

完成设计细化后，确保：
- [ ] 需求已明确并得到用户确认
- [ ] 至少考虑了2种实现方案
- [ ] 设计文档已保存到项目文档目录
- [ ] 风险已识别并有缓解措施
- [ ] 用户对设计方案满意并批准继续

## 下一步

设计批准后，应该调用 `writing-plans` 技能来创建详细的实施计划。
