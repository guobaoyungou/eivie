# 选片订单权限校验修复说明

## 修复日期
2026-03-27

## 问题现象

用户反馈：待付款的选片订单卡片，点击"详情"、"关闭订单"、"去付款"功能时，都提示"订单不存在"错误。

### 错误截图分析

1. **详情页**：显示"订单不存在"弹窗
2. **去付款页**：显示"该订单不存在"
3. **关闭订单**：操作失败

---

## 问题根本原因

### 原因分析

在 `ApiUnifiedOrder.php` 的 `detail()` 和 `closeOrder()` 方法中，查询选片订单时**缺少用户权限校验**：

#### 错误的代码（原来）❌

```php
// detail() 方法
$order = Db::name('ai_travel_photo_order')
    ->where('id', $orderId)
    ->where('aid', aid)
    ->find();  // ❌ 只校验了 aid，没有校验用户权限

// closeOrder() 方法  
$order = Db::name('ai_travel_photo_order')
    ->where('id', $orderId)
    ->where('aid', aid)
    ->where('uid', mid)  // ❌ 只用 uid 匹配，没有考虑 openid
    ->find();
```

### 为什么会出现"订单不存在"？

选片订单表 `ai_travel_photo_order` 的用户标识有两种方式：

1. **`uid` 字段**：注册用户的会员ID
2. **`openid` 字段**：H5扫码用户的微信OpenID（可能 uid=0）

**问题场景**：
- H5扫码用户下单时：`uid=0`, `openid='ox1234...'`
- 原代码只检查 `uid=mid`，但 `mid > 0`，`uid=0`
- 导致查询失败，返回"订单不存在"

---

## 解决方案

### 修复策略

参考 `getBaseWhere()` 方法中对选片订单的权限校验逻辑，添加 **uid 或 openid** 的双重匹配：

```php
// 1. 获取当前用户的 openid
$member = Db::name('member')->where('id', mid)->find();
$openid = $member['wxopenid'] ?? $member['mpopenid'] ?? '';

// 2. 构建查询
$query = Db::name('ai_travel_photo_order')
    ->where('id', $orderId)
    ->where('aid', aid);

// 3. 添加用户匹配条件（uid 或 openid）
if ($openid && mid > 0) {
    $query->where(function($q) use ($openid) {
        $q->where('uid', mid)->whereOr('openid', $openid);
    });
} else if (mid > 0) {
    $query->where('uid', mid);
} else if ($openid) {
    $query->where('openid', $openid);
}

$order = $query->find();
```

## 权限校验逻辑说明

### 匹配规则

| 场景 | uid | openid | 查询条件 |
|------|-----|--------|---------|
| 注册用户 | mid (>0) | 有值 | `uid=mid OR openid=xxx` |
| 仅注册用户 | mid (>0) | 空 | `uid=mid` |
| H5扫码用户 | 0 | 有值 | `openid=xxx` |
| 异常情况 | 0 | 空 | 无匹配条件（返回订单不存在） |

### SQL 示例

#### 注册用户（有 openid）
```sql
SELECT * FROM ai_travel_photo_order 
WHERE id = 20 
  AND aid = 1 
  AND (uid = 123 OR openid = 'ox1234abcd');
```

#### H5扫码用户（uid=0）
```sql
SELECT * FROM ai_travel_photo_order 
WHERE id = 20 
  AND aid = 1 
  AND openid = 'ox5678efgh';
```

### 参考代码

权限校验逻辑参考了 `getBaseWhere()` 方法：

```php
protected function getBaseWhere($type)
{
    $where = [];
    $where[] = ['aid', '=', aid];

    // ai_pick 表用 uid 标识用户，其他表用 mid
    if ($type === 'ai_pick') {
        $where[] = ['uid', '=', mid];
        // H5扫码用户 uid 可能为0，通过 openid 兜底匹配
        $member = Db::name('member')->where('id', mid)->find();
        if ($member) {
            // 优先匹配 wxopenid (小程序)，其次 mpopenid (公众号)
            $openid = $member['wxopenid'] ?? $member['mpopenid'] ?? '';
            if ($openid) {
                $where[] = ['openid', '=', $openid];
            }
        }
        // 由于需要 whereOr 逻辑，标记为需要特殊处理
        $where['_aipick_or'] = true;
    }
    // ...
}
```
