# 选片端演示数据说明

## 功能概述

选片模板现已集成演示数据功能，当真实数据不可用时，系统会自动使用演示数据进行展示，方便商户预览和测试选片端效果。

## 触发条件

演示数据会在以下情况自动启用：

1. **数据为空**：API接口返回的人像列表为空时
2. **网络失败**：请求失败且重试3次后仍无法获取数据时
3. **预览模式**：可用于在没有真实数据的情况下预览页面效果

## 演示数据内容

### 数据结构

系统提供5组演示人像数据：

| 组号 | 人像照片 | 生成结果数 | 二维码 |
|-----|---------|-----------|--------|
| 1 | 1张头像 | 3张照片 | ✓ |
| 2 | 1张头像 | 2张照片 | ✓ |
| 3 | 1张头像 | 3张照片 | ✓ |
| 4 | 1张头像 | 1张照片 | ✓ |
| 5 | 1张头像 | 2张照片 | ✓ |

**总计**：5个人像，11张AI生成照片

### 图片来源

演示图片使用 [Picsum Photos](https://picsum.photos) 提供的随机图片服务：

- **头像**：80x80px 随机人像
- **展示照片**：1920x1080px 随机风景
- **二维码**：示例URL链接

## 用户体验

### 演示模式提示

当系统使用演示数据时，会在页面顶部显示红色提示条：

```
演示模式：当前展示为示例数据
```

- 提示条显示5秒后自动消失
- 背景色：红色半透明
- 位置：页面顶部居中

### 自动恢复机制

使用演示数据后：

1. 系统会在60秒后自动重试获取真实数据
2. 如果真实数据可用，自动切换到真实数据展示
3. 如果仍然无数据，继续使用演示数据

## 使用场景

### 1. 商户预览

商户在配置门店时，即使尚未上传人像照片，也能预览选片端的展示效果：

```
后台 > 门店管理 > 编辑门店 > 点击"预览"按钮
→ 自动展示演示数据
```

### 2. 功能演示

向客户展示选片端功能时，无需等待真实数据生成：

```
直接访问选片URL
→ 如无数据，自动展示演示效果
```

### 3. 开发测试

前端开发和测试时，无需依赖后端真实数据：

```
本地开发环境访问
→ API请求失败时自动使用演示数据
```

## 技术实现

### 核心代码

```javascript
// 获取演示数据
getDemoData() {
    return [
        {
            id: 1,
            original_url: 'https://picsum.photos/400/600?random=1',
            thumbnail_url: 'https://picsum.photos/80/80?random=1',
            results: [
                {
                    id: 101,
                    type: 1,
                    watermark_url: 'https://picsum.photos/1920/1080?random=11'
                }
                // ...更多照片
            ],
            qrcode: 'https://www.example.com/demo1'
        }
        // ...更多人像组
    ];
}

// 使用演示数据
useDemoData() {
    this.dataList = this.getDemoData();
    this.renderContent();
    this.startPlay();
    
    // 显示提示
    // ...
    
    // 60秒后重试真实数据
    setTimeout(() => {
        this.fetchData(0);
    }, 60000);
}
```

### 触发逻辑

```javascript
// 数据为空时
if (this.dataList.length === 0) {
    this.useDemoData();
}

// 请求失败重试后
if (retryCount >= this.config.retryCount) {
    this.useDemoData();
}
```

## 注意事项

### ⚠️ 网络依赖

演示图片需要访问 `picsum.photos` 服务：

- 需要外网访问权限
- 如无法访问，图片显示失败
- 建议生产环境使用真实数据

### ⚠️ 性能考虑

每次加载会请求多张随机图片：

- 首次加载可能较慢
- 建议在有真实数据后禁用演示模式
- 图片会被浏览器缓存

### ⚠️ 数据真实性

演示数据仅用于预览：

- 不代表真实业务数据
- 二维码为示例链接，无法购买
- 投入使用前需上传真实人像

## 禁用演示数据

如需完全禁用演示数据功能，可修改代码：

```javascript
// 在 fetchData 方法中，将以下行注释或删除
// this.useDemoData();

// 改为显示空状态
this.showEmpty();
```

## 自定义演示数据
