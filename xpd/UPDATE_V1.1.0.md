# 选片端 v1.1.0 更新说明

## 📢 更新概要

本次更新为选片端添加了**演示数据支持**功能，解决了在没有真实数据时无法预览效果的问题，大幅提升了系统的可用性和用户体验。

---

## ✨ 核心新特性

### 1. 智能演示数据系统

当系统检测到以下情况时，会自动启用演示数据：

- ✅ API接口返回数据为空
- ✅ 网络请求失败且重试3次仍失败
- ✅ 新配置的门店尚未上传人像

**演示数据内容：**
- 5组演示人像
- 11张AI生成照片示例
- 完整的功能展示（轮播、头像、二维码）

### 2. 用户友好的提示机制

**页面顶部提示条：**
```
演示模式：当前展示为示例数据
```

- 红色半透明背景
- 5秒后自动消失
- 清晰标识演示状态

### 3. 自动恢复真实数据

演示模式下：
- ⏰ 60秒后自动重试获取真实数据
- 🔄 真实数据可用时无缝切换
- 🔁 失败后继续使用演示数据

## 🎯 使用场景

### 场景1：商户快速预览

**问题**：商户配置门店时想先看看效果，但还没上传照片

**解决**：
```
后台 > 门店管理 > 编辑门店 > 点击"预览"
→ 自动展示演示数据
→ 商户可立即看到完整效果
```

### 场景2：功能演示展示

**问题**：向客户演示系统功能，但没有准备测试数据

**解决**：
```
直接访问选片URL
→ 系统检测无数据
→ 自动使用演示数据
→ 完整功能演示成功
```

### 场景3：前端开发测试

**问题**：前端开发需要调试界面，但后端API不可用

**解决**：
```
本地开发环境访问
→ API请求失败
→ 自动降级到演示数据
→ 前端独立开发无阻
```

## 📊 技术实现

### 核心代码结构

```javascript
methods: {
    // 获取演示数据
    getDemoData() {
        return [
            {
                id: 1,
                original_url: 'https://picsum.photos/400/600?random=1',
                results: [...],
                qrcode: '...'
            }
            // ...5组数据
        ];
    },
    
    // 使用演示数据
    useDemoData() {
        this.dataList = this.getDemoData();
        this.renderContent();
        this.startPlay();
        
        // 显示提示
        // ...
        
        // 60秒后重试
        setTimeout(() => {
            this.fetchData(0);
        }, 60000);
    },
    
    // 数据获取（已修改）
    async fetchData(retryCount = 0) {
        try {
            // API请求...
            if (this.dataList.length === 0) {
                this.useDemoData(); // ← 新增
            }
        } catch (error) {
            if (retryCount >= this.config.retryCount) {
                this.useDemoData(); // ← 新增
            }
        }
    }
}
```

## 🔧 配置说明

### 演示数据默认配置

| 配置项 | 值 | 说明 |
|-------|---|------|
| 人像组数 | 5组 | 可修改 getDemoData() |

如需使用自己的演示图片，修改 `getDemoData()` 方法：

                    id: 101,
                    type: 1,
                    watermark_url: '你的展示图URL'
                }
            ],
            qrcode: '你的二维码内容'
        }
    ];
}

## ⚠️ 注意事项

### 1. 网络依赖

演示图片使用 `picsum.photos` 服务：
- 需要外网访问权限
- 首次加载可能较慢
- 建议生产环境使用真实数据

### 2. 数据真实性

演示数据仅供预览：
- 不代表真实业务数据
- 二维码为示例链接
- 投入使用需上传真实人像

### 3. 性能影响

演示模式下：
- 会额外加载5组随机图片
- 建议有真实数据后自动禁用
- 图片会被浏览器缓存
