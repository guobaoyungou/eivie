# 头像点击切换功能说明

## 📢 功能概述

选片端现已支持**头像点击切换**功能！用户可以直接点击下方头像列表中的任意头像，立即跳转并播放对应人像的AI生成照片。

---

## ✨ 新增特性

### 1. 交互式头像列表

**视觉反馈：**
- 🖱️ 鼠标悬停：头像放大1.1倍，边框变为红色
- 👆 点击效果：平滑切换到对应人像照片
- ✨ 动画过渡：500ms平滑过渡动画
- 🎯 高亮显示：当前播放的人像头像保持高亮

### 2. 智能播放控制

**点击头像后的行为：**
```
用户点击头像
    ↓
暂停自动播放
    ↓
跳转到对应人像的第一张照片
    ↓
更新头像高亮状态
    ↓
更新二维码显示
    ↓
500ms后恢复自动播放
```

## 🎯 使用场景

### 场景1：游客主动选择

**问题：** 游客在大屏上看到多个人像，想查看特定人像的照片

**解决：**
```
游客点击感兴趣的头像
→ 系统立即跳转到该人像照片
→ 开始轮播该人像的所有AI生成照片
→ 显示对应的二维码供扫码购买
```

### 场景2：门店人员引导

**问题：** 门店工作人员需要为游客展示特定照片

**解决：**
```
工作人员点击相应头像
→ 快速定位到目标人像
→ 展示该游客的AI照片
→ 引导扫码查看详情
```

### 场景3：快速回看

**问题：** 游客错过了某个人像的照片展示

**解决：**
```
点击之前的头像
→ 立即回到该人像的照片
→ 重新播放完整组照片
→ 不影响后续自动播放流程
```

### 核心代码

#### 1. CSS样式增强

```css
.avatar-item {
    cursor: pointer;  /* 显示手型光标 */
}

.avatar-item:hover img {
    transform: scale(1.1);      /* 悬停放大 */
    border-color: #ff6b6b;      /* 边框变红 */
}
```

#### 2. 点击事件绑定

```javascript
// 渲染头像时添加点击事件
avatarItem.addEventListener('click', () => {
    this.switchToGroup(index);
});
```

#### 3. 切换逻辑实现

```javascript
switchToGroup(groupIndex) {
    // 1. 参数验证
    if (groupIndex < 0 || groupIndex >= this.dataList.length) {
        return;
    }
    
    // 2. 暂停自动播放
    this.stopPlay();
    
    // 3. 更新状态
    this.currentGroupIndex = groupIndex;
    this.currentImageIndex = 0;
    this.groupStartTime = Date.now();
    
    // 4. 计算全局索引
    let globalIndex = 0;
    for (let i = 0; i < this.currentGroupIndex; i++) {
        globalIndex += this.dataList[i].results.length;
    }
    
    // 5. 跳转到对应位置（500ms动画）
    this.swiper.slideTo(globalIndex, 500);
    
    // 6. 更新UI
    this.updateAvatarHighlight();
    this.updateQrcode();
    
    // 7. 500ms后恢复自动播放
    setTimeout(() => {
        this.startPlay();
    }, 500);
}
```

## 💡 使用建议

### 商户培训要点

1. **告知游客可以点击**
   - 在大屏旁边放置提示标语
   - "点击头像查看您的照片"

2. **引导扫码购买**
   - 点击头像后，引导游客关注右下角二维码
   - "扫码可查看高清原图和购买"

3. **演示操作方法**
   - 工作人员主动演示点击功能
   - 增加游客互动体验

### 注意事项

⚠️ **自动播放会恢复**
- 点击后500ms会恢复自动播放
- 如需长时间停留，可连续点击同一头像

⚠️ **避免频繁点击**
- 等待切换动画完成再点击下一个
- 频繁点击可能导致状态混乱

⚠️ **移动端优化**
- 在触摸屏设备上同样生效
- 确保点击区域足够大（80px）
