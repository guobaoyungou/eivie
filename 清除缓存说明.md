# 清除缓存说明

## 问题现象

修改后端代码后，前端页面仍然显示旧数据（"待发货"而不是"已完成"）。

## 可能原因

1. **PHP OpCache 缓存**：PHP 缓存了旧的代码
2. **浏览器缓存**：浏览器缓存了旧的 API 响应数据

## 解决方案

### 方案1：清除 PHP OpCache（推荐）

#### 方法1：重启 PHP-FPM 服务

```bash
# 重启 PHP-FPM（根据您的系统版本选择）
systemctl restart php-fpm
# 或
systemctl restart php7.4-fpm
# 或
service php-fpm restart
```

#### 方法2：通过脚本清除（临时）

创建文件 `/home/www/ai.eivie.cn/public/clear_cache.php`：

```php
<?php
// 清除 OpCache
if (function_exists('opcache_reset')) {
    opcache_reset();
    echo "OpCache cleared successfully!\n";
} else {
    echo "OpCache is not enabled.\n";
}

// 显示当前时间
echo "Time: " . date('Y-m-d H:i:s') . "\n";
?>
```

然后访问：`https://ai.eivie.cn/clear_cache.php`

访问后删除该文件：
```bash
rm /home/www/ai.eivie.cn/public/clear_cache.php
```

### 方案2：清除浏览器缓存

#### Chrome/Edge 浏览器

1. 打开开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

或者：

1. 按 `Ctrl + Shift + Delete`（Mac: `Cmd + Shift + Delete`）
2. 选择"缓存的图片和文件"
3. 时间范围选择"全部时间"
4. 点击"清除数据"

#### 手动清除 API 缓存

1. 打开开发者工具（F12）
2. 切换到 Network 面板
3. 勾选"Disable cache"（禁用缓存）
4. 刷新页面

### 方案3：强制刷新前端

如果使用了 UniApp H5 编译：

```bash
cd /home/www/ai.eivie.cn/uniapp
npm run build:h5
```

### 方案4：临时添加版本参数

修改前端 API 调用，添加时间戳参数强制刷新：

```javascript
// 在 app.post 调用时添加时间戳
app.post('ApiUnifiedOrder/orderlist?_t=' + Date.now(), {...}, function(res){
    // ...
});
```

## 验证方法

### 1. 直接访问 API 接口

在浏览器中访问（需要先登录）：

```
https://ai.eivie.cn/index.php?s=/api/ApiUnifiedOrder/orderlist&st=3&pagenum=1&_t=123456
```

查看返回的 JSON 数据中，`status_text` 字段是否为"已完成"。

### 2. 检查 Network 响应

1. 打开开发者工具（F12）
2. 切换到 Network 面板
3. 刷新订单列表页
4. 找到 `ApiUnifiedOrder/orderlist` 请求
5. 查看 Response 标签页
6. 检查 `datalist[].status_text` 是否为"已完成"

### 3. 控制台输出数据

在订单列表页的控制台执行：

```javascript
// 查看订单数据
console.log(this.datalist);
```

检查订单对象的 `status_text` 字段值。

## 注意事项

1. 如果是生产环境，重启 PHP-FPM 会短暂影响服务，建议选择低峰期操作
2. 清除缓存后，首次访问可能会稍慢（因为需要重新编译 PHP 代码）
3. 如果使用了 CDN，可能还需要清除 CDN 缓存
