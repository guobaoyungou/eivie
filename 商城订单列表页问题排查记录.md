# 商城订单列表页问题排查记录

## 问题描述

访问 `https://ai.eivie.cn/h5/1.html#/pagesExt/order/orderlist?st=0` 显示空白，控制台报错：
```
TypeError: Cannot read properties of undefined (reading 'length')
```

## 排查步骤

### 1. 接口调用分析

**问题页面**：`/pagesExt/order/orderlist`  
**调用接口**：`ApiOrder/orderlist`  
**请求参数**：
```javascript
{
  st: '0',
  pagenum: 1,
  keyword: ''
}
```

### 2. 错误原因

从错误堆栈分析，问题出在：
```javascript
var data = res.datalist;
if (data.length == 0) {  // 这里报错，说明 data 是 undefined
```

说明接口返回的 `res.datalist` 可能是 `undefined`，而不是数组。

### 3. 可能的原因

#### 原因1：接口返回格式不符合预期
- 后端接口可能返回了错误信息，而不是正常的数据格式
- 可能返回了 `{status: 0, msg: '错误信息'}` 而没有 `datalist` 字段

#### 原因2：接口权限问题
- 用户可能未登录或登录已过期
- 接口需要验证但验证失败

#### 原因3：后端代码执行异常
- 数据库查询出错
- 某个变量未定义导致PHP报错

### 4. 已实施的修复

#### 修复1：添加数据校验（第一次）
```javascript
var data = res.datalist || [];
```
- 即使 `res.datalist` 是 `undefined`，也会使用空数组
- 避免读取 `length` 属性时报错

#### 修复2：添加完整的错误处理（第二次）
```javascript
if(!res || typeof res !== 'object'){
  console.error('接口返回数据格式错误', res);
  that.nodata = true;
  that.loaded();
  return;
}
var data = res.datalist || [];
```
- 检查响应对象是否存在
- 检查响应对象是否是对象类型
- 记录错误日志到控制台
- 显示空数据提示

## 11. 问题根本原因（最终确认）

### 真实错误

用户提供的 HTML 错误页面显示了真正的错误信息：

```
SQLSTATE[42S22]: Column not found: 1054 Unknown column 'goods_id' in 'field list'
```

**ThinkPHP 6.0.7 错误页面**

### 错误定位

错误发生在 `/home/www/ai.eivie.cn/app/controller/ApiUnifiedOrder.php` 文件中：

**问题代码**（3处）：

1. **第 172 行** - `getAiPickOrderDetail()` 方法
```php
foreach ($goodsList as $goods) {
    $prolist[] = [
        'proid' => $goods['goods_id'],  // ❌ 错误
        'num' => $goods['quantity'] ?? 1,  // ❌ 错误
    ];
}
```

2. **第 877 行** - `normalizeAiPickOrder()` 方法的查询
```php
$goodsList = Db::name('ai_travel_photo_order_goods')
    ->where('order_id', $row['id'])
    ->field('id,order_id,goods_id,goods_name,goods_image,quantity,price')  // ❌ 错误字段
    ->select()->toArray();
```

3. **第 912 行** - `normalizeAiPickOrder()` 方法的数据映射
```php
foreach ($goodsList as $goods) {
    $prolist[] = [
        'proid' => $goods['goods_id'],  // ❌ 错误
        'num' => $goods['quantity'] ?? 1,  // ❌ 错误
    ];
}
```

### 数据表实际字段

查看 `ai_travel_photo_order_goods` 表结构：

```sql
CREATE TABLE `ddwx_ai_travel_photo_order_goods` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `order_id` int(11) NOT NULL,
  `result_id` int(11) NOT NULL,      -- ✅ 正确字段（不是 goods_id）
  `goods_name` varchar(255),
  `goods_image` varchar(500),
  `price` decimal(10,2),
  `num` int(11) DEFAULT 1,           -- ✅ 正确字段（不是 quantity）
  PRIMARY KEY (`id`)
);
```

**关键发现**：
- ❌ 表中**没有** `goods_id` 字段
- ❌ 表中**没有** `quantity` 字段
- ✅ 应该使用 `result_id` 字段
- ✅ 应该使用 `num` 字段

### 错误原因分析

这个错误实际上与商城订单列表页**无关**，而是：

1. **用户访问的是商城订单列表页**：`/pagesExt/order/orderlist?st=0`
2. **该页面调用的应该是商城订单接口**：`ApiOrder/orderlist`
3. **但之前可能被错误修改为调用**：`ApiUnifiedOrder/orderlist`
4. **统一订单接口会聚合所有订单类型**，包括 AI 旅拍选片订单
5. **查询选片订单时使用了错误的字段名**，导致 SQL 错误
