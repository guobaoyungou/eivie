```
│        MySQL + Cache(Redis) + SSE实时推送              │
└──────────────────────────────────────────────────────┘
```

### 核心优势
| 维度 | 实现方案 | 效果 |
|------|---------|------|
| **极简操作** | 活动模板向导 + 扫码即用 | 非技术人员10分钟上手 |
| **低延迟+高稳定** | SSE实时推送 + 缓存层 + 增量拉取 | 万人并发不卡顿 |
| **视觉可定制** | CSS变量主题 + 7种动画预设 + 品牌LOGO | 完全自定义外观 |
| **内容安全可控** | 三层过滤链 + 一键禁言 + 人工审核 | 防广告/违规零漏洞 |

### 3.5 内容安全管理⭐

```
进入「内容安全」标签
        │
        ├── 🛡️ 安全设置
        │     ├── 内容过滤：开启/关闭
        │     ├── 人工审核：关闭(直接显示) / 开启(先审后显)
        │     ├── 最大消息长度：10~500字
        │     ├── 发送间隔：0~60秒
        │     └── [一键全局禁言] 按钮 ← 紧急情况立即生效
        │
        ├── 🚫 关键词过滤
        │     ├── 添加关键词
        │     │     ├── 关键词内容
        │     │     ├── 匹配方式：包含/完全/正则
        │     │     └── 命中动作：替换为***/拒绝发送/转人工审核
        │     ├── 批量添加（每行一个或逗号分隔）
        │     ├── 启用/禁用单个关键词
        │     └── 删除关键词
        │     📌 系统已内置基础敏感词库（赌博/色情/广告等20+词）
        │
        └── 🚷 用户禁言
              ├── 添加禁言
              │     ├── 用户标识(openid)
              │     ├── 昵称(可选)
              │     ├── 禁言原因
              │     └── 时长：留空=永久 / 输入分钟数=定时
              └── 解除禁言
```

**内容过滤链路（自动执行）**：
```
用户发送消息/弹幕
    │
    ├── ① 全局禁言检查 → 已开启? → 拒绝: "当前活动已开启全局禁言"
    │
    ├── ② 用户禁言检查 → 在黑名单? → 拒绝: "您已被禁言"
    │
    ├── ③ 自定义关键词过滤
    │     ├── 包含匹配 → 替换为*** / 拒绝 / 转审核
    │     ├── 完全匹配 → 同上
    │     └── 正则匹配 → 同上
    │
    ├── ④ 内置敏感词库 → 自动替换为***
    │
    └── ⑤ 通过 → 发送成功 / 进入审核队列
```

## 5. 现场大屏操作流程

### 5.1 大屏展示

```
现场操作员在电脑上打开 wxhd.eivie.cn/s/{access_code}
        │
        ▼
  大屏页面自动加载
        │
        ├── 顶部: 品牌LOGO + 活动名称 + 状态
        ├── 功能导航栏: 可切换不同功能展示
        ├── 内容展示区: 签到/抽奖/投票/弹幕...
        ├── 二维码区域: 参与者扫码入口
        └── 背景动画: 根据品牌定制设置自动播放
```

### 5.2 实时数据推送（SSE）⭐

```
大屏页面加载后自动建立 SSE 连接
        │
        ├── 频道订阅: sign, wall, danmu, vote, lottery, shake, notice
        │
        ├── 数据更新流程:
        │     参与者操作 → 后端处理 → 通知SSE频道 → 大屏自动刷新
        │     延迟: < 1秒（无需手动刷新）
        │
        ├── 断线重连:
        │     连接断开 → 自动重连(最多5次) → 指数退避 → 降级为轮询
        │
        └── 心跳保活:
              每8秒发送心跳 → 防止连接超时断开
```

**SSE 频道说明**：

| 频道 | 推送时机 | 数据内容 |
|------|---------|---------|
| `sign` | 新用户签到时 | 最新20个签到+总数 |
| `wall` | 新消息上墙时 | 最新30条上墙消息 |
| `danmu` | 新弹幕发送时 | 最近5秒内弹幕 |
| `vote` | 新投票时 | 各选项票数+总票数 |
| `lottery` | 抽奖完成时 | 中奖者信息 |
| `shake` | 分数提交时 | 排行榜top20 |
| `notice` | 管理员发公告时 | 公告内容 |

## 7. 系统技术特性

### 7.1 低延迟架构

| 技术 | 作用 | 性能指标 |
|------|------|---------|
| **SSE (Server-Sent Events)** | 替代轮询，服务端主动推送 | 延迟 < 1秒 |
| **Cache 缓存层** | 高频查询缓存（签到列表、投票统计、安全配置） | 减少80%+ DB查询 |
| **增量拉取** | 签到/消息使用 `last_id` 参数增量获取 | 减少数据传输量 |
| **连接池复用** | ThinkPHP 6 + PHP-FPM连接复用 | 高并发处理能力 |

### 7.2 安全架构

```
三层安全过滤链:
┌─────────────────────────────────┐
│  第1层: 全局禁言 + 用户禁言       │  ← 管理员一键控制
├─────────────────────────────────┤
│  第2层: 自定义关键词过滤          │  ← 支持包含/完全/正则匹配
│         动作: 替换*** / 拒绝 / 审核 │
├─────────────────────────────────┤
│  第3层: 内置敏感词库              │  ← 20+ 高频违禁词自动过滤
│         赌博/色情/广告/诈骗...      │
└─────────────────────────────────┘
```

### 7.3 多租户隔离

```
数据隔离: aid(平台) + bid(商家) + activity_id(活动) 三级隔离
所有数据操作自动注入租户标识，确保数据安全
```

### 7.4 CSS 变量主题系统

```css
:root {
  --hd-primary: #6366f1;      /* 主色调 */
  --hd-secondary: #8b5cf6;    /* 辅色调 */
  --hd-accent: #f59e0b;       /* 强调色 */
  --hd-bg: #0a0a1a;           /* 背景色 */
```

## 8. 常见问题（FAQ）

### Q1: 创建活动最快需要多久？
**A**: 使用模板向导，**30秒**即可完成创建。选择模板 → 输入名称 → 保存，功能自动配置。

### Q2: 如何让观众参与活动？
**A**: 将活动二维码投放到现场（打印/投影），观众微信扫码即可参与。**零下载、零注册、扫码即用**。

### Q3: 活动进行中出现违规消息怎么办？
**A**: 三种应急方案：
1. **一键全局禁言** — 立即禁止所有消息发送
2. **禁言用户** — 精准封禁违规用户
3. **关键词过滤** — 自动拦截违规内容

### Q4: 大屏数据多久更新一次？
**A**: 使用 SSE 实时推送，数据变化后 **< 1秒** 自动更新到大屏，无需手动刷新。

### Q5: 如何自定义活动视觉效果？
**A**: 进入「品牌定制」标签：
1. 上传品牌LOGO → 大屏顶部展示
2. 选择主题色 → 全局颜色自动切换
3. 设置背景 → 纯色/渐变/图片/视频
4. 选择动画风格 → 7种预设任选

### Q6: 支持多少人同时参与？
**A**: 系统设计支持万人级并发：
- SSE 实时推送减轻轮询压力
- Cache 缓存层减少数据库查询
- 增量数据拉取减少带宽消耗

### Q7: 活动可以复用/克隆吗？
**A**: 支持活动克隆功能，一键复制活动配置到新活动。
