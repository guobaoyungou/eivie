# AI旅拍系统设置功能 - 快速使用指南

## 访问路径
后台管理系统 → AI旅拍 → 系统设置

## 功能概览

### Tab1: 基础设置
**用途**: 配置AI旅拍的基本业务参数

#### 配置项
- ✅ **启用AI旅拍功能**: 开关控制整个AI旅拍功能
- 💰 **价格设置**: 
  - 单张图片价格 (默认9.9元)
  - 单个视频价格 (默认29.9元)
- 🖼️ **水印设置**:
  - Logo水印图片上传
  - 水印位置选择 (右下/左下/右上/左上)
- 📱 **二维码设置**: 有效期 (1-365天，默认30天)
- 🎬 **视频设置**:
  - 自动生成视频开关
  - 默认时长 (5秒/10秒)
- 🎨 **场景设置**: 最大生成场景数 (1-50，默认10)

---

### Tab2: OSS配置
**用途**: 配置商家独立的阿里云OSS存储

#### 使用场景
- 商家需要使用独立的OSS存储
- 商家有自己的CDN加速域名
- 商家需要独立的存储管理

#### 配置项
- AccessKey ID: 阿里云OSS访问密钥ID
- AccessKey Secret: 阿里云OSS访问密钥Secret
- Bucket名称: OSS存储桶名称
- Endpoint: OSS服务地址 (如: oss-cn-hangzhou.aliyuncs.com)
- CDN域名 (可选): 自定义CDN域名

#### 注意事项
⚠️ **为空时使用平台统一OSS配置，填写后使用商家独立配置**

---

### Tab3: API密钥管理 ⭐核心功能
**用途**: 管理多个AI模型API Key，实现轮询和并发优化

#### 为什么需要多个Key？
- **通义万相**: 单Key限制5个并发/秒
- **可灵AI**: 单Key限制3个并发/秒
- **解决方案**: 配置多个Key轮询使用，突破并发限制

#### 通义万相API密钥 (图生图)
**建议配置**: 3-5个Key

##### 操作步骤
1. 点击"添加通义万相Key"按钮
2. 填写API Key (必填)
3. 填写API Secret (可选)
4. 设置最大并发数 (建议5个)
5. 设置单张图片成本 (默认0.05元)
6. 选择是否启用
7. 设置为默认Key (可选)
8. 点击"测试连接"验证Key有效性
9. 保存

##### 表格说明
- **API Key**: 脱敏显示，仅显示前6位和后4位
- **状态**: 启用/禁用
- **并发**: 当前并发数/最大并发数 (如: 2/5)
- **调用统计**: 总调用次数
- **成功率**: 成功调用百分比
- **总成本**: 累计消耗金额
- **最后调用**: 最后一次调用时间

#### 可灵AI API密钥 (图生视频)
**建议配置**: 2-4个Key

操作步骤同通义万相，最大并发数建议3个。

#### Key轮询机制
系统自动按以下规则选择Key:
1. 查询商家启用的Key列表
2. 如果商家没有配置，使用平台默认Key
3. 获取各Key当前并发数
4. 按并发数升序排列
5. 选择第一个未达并发上限的Key
6. 如果所有Key都达上限，任务加入队列等待

#### 使用建议
✅ 每个模型至少配置2个以上Key保证高可用  
✅ Key达到并发上限时自动切换到下一个可用Key  
✅ 系统自动按当前并发数选择最空闲的Key  
✅ 可在统计页面查看各Key的详细使用情况  

---

### Tab4: 队列配置
**用途**: 配置Redis队列的并发数和超时时间

#### 抠图队列
- 并发数: 1-100 (默认10)
- 超时时间: 10-3600秒 (默认120秒)

#### 图生图队列
- 并发数: 1-100 (默认5)
- 超时时间: 10-3600秒 (默认180秒)

#### 图生视频队列
- 并发数: 1-100 (默认3)
- 超时时间: 10-3600秒 (默认600秒)

#### 配置说明
⚠️ **并发数不宜设置过大，建议根据API Key数量调整**  
⚠️ **超时时间应大于AI接口的平均响应时间**  
⚠️ **修改后需重启队列Worker才能生效**  

---

### Tab5: 监控告警
**用途**: 配置系统监控阈值和告警通知

#### 告警阈值
- **队列积压阈值**: 当前排队任务数超过此值时告警 (默认1000)
- **失败率阈值**: 近1小时失败率超过此值时告警 (默认5%)
- **响应时间阈值**: AI接口平均响应时间超过此值时告警 (默认90秒)

#### 告警邮箱
- 支持配置多个邮箱
- 一行一个邮箱地址
- 示例:
  ```
  admin@example.com
  tech@example.com
  ```

## 常见问题

### Q1: 为什么需要配置多个API Key？
**A**: 单个API Key有并发限制（通义万相5个/秒，可灵AI 3个/秒），配置多个Key可以突破这个限制，提高系统并发处理能力。

### Q2: 如何确定需要配置几个Key？
**A**: 根据业务量评估。假设高峰期每秒需要处理15个图生图请求，至少需要3个通义万相Key（15/5=3）。

### Q3: 商家配置的OSS和API Key与平台配置的关系？
**A**: 商家配置优先级更高。商家配置为空时，自动使用平台配置。商家可根据需要使用独立的OSS和API Key。

### Q4: Key被禁用后怎么办？
**A**: 系统会自动将该Key标记为禁用状态并发送告警通知，管理员需要检查Key是否过期或欠费，解决后重新启用。

### Q5: 如何测试API Key是否有效？
**A**: 在API Key编辑弹窗中，点击"测试连接"按钮，系统会调用AI接口测试Key的有效性并返回响应时间。

### Q6: 修改队列配置后需要重启吗？
**A**: 是的，队列配置（并发数、超时时间）需要重启队列Worker才能生效。其他配置无需重启。

### Q7: 如何查看各个Key的使用情况？
**A**: 在API密钥管理Tab的列表中，可以看到每个Key的实时并发、总调用次数、成功率、总成本等统计数据。

## 最佳实践

### 1. API Key配置建议
- **通义万相**: 配置3-5个Key
- **可灵AI**: 配置2-4个Key
- 至少保证每个模型有2个以上Key
- 定期检查Key的有效性和余额

### 2. 队列配置建议
- 并发数 = API Key数量 × 单Key并发限制
- 例如: 3个通义万相Key × 5个并发 = 15个并发
- 超时时间建议设置为平均响应时间的2-3倍

### 3. 监控告警建议
- 队列积压阈值: 根据日常业务量设置，建议为日均量的10%
- 失败率阈值: 5-10%之间，过低会频繁告警
- 响应时间阈值: 根据AI接口的SLA设置

### 4. 成本控制建议
- 定期查看各Key的总成本统计
- 根据成本数据优化价格策略
- 监控异常高成本的Key，可能存在问题
