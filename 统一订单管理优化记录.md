## 优化 1：已付款的选片订单放在已完成

### 问题描述
AI 旅拍选片订单有两个状态：
- `status=0`：待支付
- `status=1`：已付款

之前的逻辑将 `status=1` 映射到"待收货"，但选片订单没有发货和收货的概念，应该直接归类到"已完成"。

### 解决方案

**文件**：`/home/www/ai.eivie.cn/app/controller/ApiUnifiedOrder.php`

**修改位置 1**：`toUnifiedStatus()` 方法（约第 546 行）

```php
// 修改前 ❌
if ($type === 'ai_pick') {
    if ($actualStatus == 0) return 0;
    if ($actualStatus == 1) return 2;  // 映射到待收货
    return 3;
}

// 修改后 ✅
if ($type === 'ai_pick') {
    if ($actualStatus == 0) return 0;
    if ($actualStatus == 1) return 3;  // 映射到已完成
    return 3;
}
```

**修改位置 2**：`mapStatus()` 方法（约第 510 行）

```php
// 修改前 ❌
if ($type === 'ai_pick') {
    $map = [
        0 => 0,
        1 => null,
        2 => 1,      // 待收货 → status=1
        3 => null,
        4 => [3, 4],
    ];
}

// 修改后 ✅
if ($type === 'ai_pick') {
    $map = [
        0 => 0,
        1 => null,
        2 => null,   // 选片无待收货
        3 => 1,      // 已完成 → status=1
        4 => [3, 4],
    ];
}
```

**修改位置 3**：`countWithStatus()` 方法（约第 1026 行）

```php
// 修改前 ❌
if ($type === 'ai_pick') {
    if ($st == '0') {
        $where[] = ['status', '=', 0];
    } elseif ($st == '2') {
        $where[] = ['status', '=', 1];  // 待收货
    } else {
        return 0;
    }
}

// 修改后 ✅
if ($type === 'ai_pick') {
    if ($st == '0') {
        $where[] = ['status', '=', 0];
    } elseif ($st == '3') {
        $where[] = ['status', '=', 1];  // 已完成
    } else {
        return 0;
    }
}
```

### 效果
- ✅ 待付款的选片订单显示在"待付款"标签
- ✅ 已付款的选片订单显示在"已完成"标签
- ✅ 订单数量统计正确

## 优化 3：待付款订单点击"去付款"正常跳转

### 问题描述
用户点击选片订单的"去付款"按钮时，提示"该订单不存在"，无法正常跳转到支付页面。

### 原因分析
1. 订单卡片组件缺少"去付款"按钮
2. 后端返回的订单数据中缺少 `payorderid` 字段
3. `ApiPay.php` 中没有处理 `ai_pick` 类型订单的跳转逻辑

### 解决方案

#### 1. 前端添加"去付款"按钮

**文件**：`/home/www/ai.eivie.cn/uniapp/components/unified-order-card/unified-order-card.vue`

```vue
<!-- 操作区 -->
<view class="card-op">
    <!-- 待付款状态显示去付款按钮 -->
    <view v-if="item.status == 0 && item.payorderid" 
          class="btn-pay" 
          :style="{background: themeColor}" 
          @tap.stop="goPay">去付款</view>
    <view class="btn-detail" 
          :style="{background: themeColor}" 
          @tap.stop="goDetail">详情</view>
</view>
```

添加 `goPay` 方法：

```javascript
methods: {
    goPay: function() {
        if (this.item.payorderid) {
            app.goto('/pagesExt/pay/pay?id=' + this.item.payorderid);
        }
    }
}
```

#### 2. 后端返回 `payorderid` 字段

**文件**：`/home/www/ai.eivie.cn/app/controller/ApiUnifiedOrder.php`

在 `normalizeAiPickOrder()` 方法中添加查询支付订单ID的逻辑：

## 优化 4：显示 50x50px 的缩略图

### 问题描述
选片订单的商品图片没有使用缩略图，直接加载原图导致列表加载缓慢。

### 解决方案

**文件**：`/home/www/ai.eivie.cn/app/controller/ApiUnifiedOrder.php`

在 `normalizeAiPickOrder()` 方法中添加图片缩略图处理：

```php
// 处理封面图缩略图
if ($coverImage && strpos($coverImage, 'http') === 0) {
    // 添加100x100缩略图参数（适配阿里云OSS、腾讯云COS等）
    if (strpos($coverImage, '?') === false) {
        $coverImage = $coverImage . '?x-oss-process=image/resize,w_100,h_100';
    }
}

// 处理商品图片缩略图
foreach ($goodsList as $goods) {
    $goodsPic = $goods['goods_image'] ?? $coverImage;
    if ($goodsPic && strpos($goodsPic, 'http') === 0) {
        if (strpos($goodsPic, '?') === false) {
            $goodsPic = $goodsPic . '?x-oss-process=image/resize,w_100,h_100';
        }
    }
    
    $prolist[] = [
        // ... 其他字段
        'pic' => $goodsPic,  // 使用缩略图
    ];
}
```

### 技术说明

1. **缩略图参数**：`?x-oss-process=image/resize,w_100,h_100`
   - 适用于阿里云 OSS
   - 腾讯云 COS 可能需要调整为：`?imageMogr2/thumbnail/100x100`
   - 其他 CDN 根据实际情况调整

2. **尺寸选择**：100x100px（实际显示50x50px，2倍图保证清晰度）
   - 前端样式：`width: 100rpx; height: 100rpx;`（相当于 50x50px）
   - 后端缩略图：100x100px（2倍图）

3. **性能优化**：
   - 原图可能是几MB，缩略图只有几KB
   - 大幅提升列表加载速度
   - 减少流量消耗

### 效果
- ✅ 订单列表封面图使用缩略图
- ✅ 商品清单中的商品图片使用缩略图
- ✅ 列表加载速度明显提升
- ✅ 图片清晰度满足显示需求
