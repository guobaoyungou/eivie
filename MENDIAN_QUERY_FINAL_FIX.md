# 门店查询逻辑最终修复方案

## 问题分析

### 用户需求
商家登录后，在人像上传弹窗中应该只看到**自己商家的门店**，而不是所有门店。

### 数据库现状
```sql
SELECT id, name, bid FROM mendian WHERE aid=1;

结果：
- 绿万鸿花卉园艺 (bid=0)  ← 历史公共数据
- 木森林园艺 (bid=0)       ← 历史公共数据  
- 瑾上添花 (bid=0)         ← 历史公共数据
- 谭会 (bid=10)           ← 商家10的门店
- 贵州特产 (bid=11)        ← 商家11的门店
- ... 等等
```

### 业务逻辑
1. **bid=0的门店** 是历史公共数据，所有商家都应该能看到和使用
2. **bid>0的门店** 是具体商家的专属门店，只有该商家能看到

## 最终解决方案

### 查询逻辑
```php
// 获取门店列表（获取当前商家的门店 + bid=0的公共门店）
$mendian_list = Db::name('mendian')
    ->where('aid', $this->aid)
    ->where(function($query) {
        // 获取当前商家的门店（bid匹配）或bid=0的公共门店
        $query->whereOr([
            ['bid', '=', $this->bid],
            ['bid', '=', 0]
        ]);
    })
    ->select();
```

### 查询效果

**商家A (bid=10) 看到的门店**:
```
- 绿万鸿花卉园艺 (bid=0)  ← 公共门店
- 木森林园艺 (bid=0)       ← 公共门店
- 瑾上添花 (bid=0)         ← 公共门店
- 谭会 (bid=10)           ← 自己的门店
```

**商家B (bid=11) 看到的门店**:
```
- 绿万鸿花卉园艺 (bid=0)  ← 公共门店
- 木森林园艺 (bid=0)       ← 公共门店
- 瑾上添花 (bid=0)         ← 公共门店
- 贵州特产 (bid=11)        ← 自己的门店
```

**超级管理员 (bid=0) 看到的门店**:
```
- 绿万鸿花卉园艺 (bid=0)  ← 公共门店
- 木森林园艺 (bid=0)       ← 公共门店
- 瑾上添花 (bid=0)         ← 公共门店
```

## 优势

1. ✅ **权限隔离**: 各商家只能看到自己的门店，看不到其他商家的门店
2. ✅ **历史兼容**: 兼容bid=0的历史公共门店数据
3. ✅ **业务合理**: 公共门店所有商家都能使用，专属门店只有所属商家能用
4. ✅ **无需迁移**: 不需要修改数据库中的历史数据

## 技术说明

使用ThinkPHP的闭包查询构建OR条件：
```php
->where(function($query) {
    $query->whereOr([
        ['bid', '=', $this->bid],  // 自己的门店
        ['bid', '=', 0]            // 或公共门店
    ]);
})
```

等价SQL:
```sql
WHERE aid = ? AND (bid = ? OR bid = 0)
```
