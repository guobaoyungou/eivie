# 选片订单成片状态优化说明

## 优化日期
2026-03-27

## 优化需求

已完成状态下的选片订单，需要根据成片状态显示不同的操作按钮：
- **成片存在且未被删除** → 显示"去下载"按钮
- **成片已过期或被删除** → 显示"已过期"状态（灰色按钮，不可点击）

## 技术实现

### 1. 后端逻辑（ApiUnifiedOrder.php）

**文件路径**：`/home/www/ai.eivie.cn/app/controller/ApiUnifiedOrder.php`

#### 修改位置：`normalizeAiPickOrder()` 方法

在查询支付订单ID后，添加成片状态检查逻辑：

```php
// 查询成片状态（用于显示"去下载"或"已过期"）
$resultStatus = 'normal';  // normal=正常, expired=已过期
$downloadUrl = '';
if ($actualStatus == 1 && !empty($goodsList)) {
    // 已付款订单，检查成片是否存在且未被删除
    $resultIds = array_column($goodsList, 'result_id');
    $resultIds = array_filter($resultIds);  // 过滤掉空值
    
    if (!empty($resultIds)) {
        // 查询成片状态（status=1为正常，status=2为已删除）
        $validResultCount = Db::name('ai_travel_photo_result')
            ->where('id', 'in', $resultIds)
            ->where('status', 1)  // 只统计未被删除的成片
            ->count();
        
        // 如果所有成片都被删除了，标记为已过期
        if ($validResultCount == 0) {
            $resultStatus = 'expired';
        } else {
            // 有成片存在，生成下载链接（指向订单详情页）
            $resultStatus = 'normal';
            $downloadUrl = '/pagesExt/order/detail?id=' . $row['id'];
        }
    } else {
        // 没有关联成片，标记为已过期
        $resultStatus = 'expired';
    }
}
```

#### 返回数据结构

在订单数据中新增两个字段：

```php
return [
    // ... 其他字段
    'result_status' => $resultStatus,  // 成片状态：normal=正常, expired=已过期
    'download_url'  => $downloadUrl,   // 下载链接
];
```

## 数据表说明

### ai_travel_photo_result 表

成片结果表的 `status` 字段定义：

| 值 | 含义 | 说明 |
|----|------|------|
| 0 | 禁用 | 成片被禁用 |
| 1 | 正常 | 成片存在且可用 |
| 2 | 已删除 | 成片已被删除 |

**判断逻辑**：
- 只有 `status=1` 的成片才视为有效
- `status=2` 的成片视为已过期

---

## 业务逻辑流程

```
已完成的选片订单（status=1）
    ↓
查询订单关联的成片ID（result_id）
    ↓
检查成片表中的状态（ai_travel_photo_result.status）
    ↓
    ├─ 有成片且 status=1  → result_status='normal' → 显示"去下载"按钮
    └─ 无成片或 status=2  → result_status='expired' → 显示"已过期"按钮
```
