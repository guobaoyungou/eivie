# 订单卡片显示订单号和下单日期优化说明

## 修复时间
2026-03-27

## 问题描述

用户反馈：订单列表卡片的顶部应该显示订单号和下单日期，但现在只有订单号，且订单号显示位置不明显。

### 截图问题分析

从用户提供的截图可以看出：
- 订单卡片顶部只显示了"选片"标签和"待付款"状态
- 订单号和下单日期的位置是空白的（红框标注处）
- 缺少订单的基本信息展示

## 问题原因

前端组件 `unified-order-card.vue` 的布局设计有问题：
1. 订单号 `ordernum` 直接放在标签后面，容易被省略号截断
2. 没有显示下单日期 `create_time` 字段
3. 订单号颜色为灰色 `#999`，不够醒目

## 修复内容

### 前端组件优化

**文件位置**：`/home/www/ai.eivie.cn/uniapp/components/unified-order-card/unified-order-card.vue`

#### 1. 模板结构调整（第 3-13 行）

**修复前**：
```vue
<view class="card-head">
    <view class="head-left">
        <text class="type-tag">{{item.order_type_name}}</text>
        <text class="ordernum">{{item.ordernum}}</text>
    </view>
    <text class="status-text">{{item.status_text}}</text>
</view>
```

**修复后**：
```vue
<view class="card-head">
    <view class="head-left">
        <text class="type-tag">{{item.order_type_name}}</text>
        <view class="order-info">
            <text class="ordernum">{{item.ordernum}}</text>
            <text class="create-time" v-if="item.create_time">{{item.create_time}}</text>
        </view>
    </view>
    <text class="status-text">{{item.status_text}}</text>
</view>
```

**优化说明**：
- 新增 `order-info` 容器，采用垂直布局（`flex-direction: column`）
- 订单号放在第一行
- 下单日期放在第二行
- 使用 `v-if="item.create_time"` 确保有数据时才显示

#### 2. 样式优化

**修复前**：

**修复后**：
```css

    display: flex;

    flex: 1;
    overflow: hidden;
}
.type-tag {
    font-size: 20rpx;

```

**优化说明**：
- 新增 `.order-info` 样式，垂直排列订单号和日期
- 订单号颜色从 `#999`（灰色）改为 `#333`（深色），更加醒目
- 订单号字体大小 `24rpx`，行高 `32rpx`
- 下单日期字体大小 `20rpx`（略小），行高 `28rpx`，颜色 `#999`（灰色）
- 两者都使用省略号处理超长文本

## 后端数据确认

后端 `ApiUnifiedOrder.php` 的 `normalizeAiPickOrder()` 方法已经返回了所需字段：

```php
return [
    'ordernum'         => $row['order_no'] ?? '',         // 订单号
    'create_time'      => $timestamp ? date('Y-m-d H:i', $timestamp) : '',  // 下单日期
    // ... 其他字段
];
```

**数据格式**：
- `ordernum`：订单号，格式如 `AIPICK20260327123456`
- `create_time`：下单日期，格式如 `2026-03-27 14:30`
